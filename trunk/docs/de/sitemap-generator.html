Bevor Sie beginnen

Der Google Sitemap-Generator ist ein Python-Skript, das mit dem Sitemap-Protokoll eine Sitemap für Ihre Website erstellt. Dieses Skript kann Sitemaps aus URL-Listen, Webserver-Verzeichnissen oder Zugriffsprotokollen erstellen. Zur Verwendung dieses Skripts:

  • Sie müssen eine Verbindung zum Webserver herstellen und Skripts ausführen können.
  • Auf Ihrem Webserver muss Python 2.2 oder höher installiert sein.
  • Sie müssen den Befehl kennen, mit dem Sie Python starten. (In der Regel ist dies python, das hängt allerdings von der Installation ab. Sind auf Ihrem Webserver beispielsweise zwei Versionen von Python installiert, kann die ältere Version mit dem Befehl python und die neuere Version mit python2 aufgerufen werden.)
  • Sie müssen den Verzeichnispfad zu Ihrer Website kennen. Wenn Ihr Webserver für nur eine Website betrieben wird, kann dies ein Pfad wie der folgende sein: var/www/html. Wenn Sie einen virtuellen Server haben, der für mehrere Websites betrieben wird, kann dies ein Pfad wie der folgende sein: home/virtual/site1/fst/var/www/html.
  • Sie müssen Dateien auf Ihren Webserver hochladen können (zum Beispiel mit FTP).
  • Wenn Sie eine Liste von URLs auf der Grundlage von Zugriffsprotokollen erstellen möchten, müssen Sie die für diese Protokolle verwendete Verschlüsselung sowie den vollständigen Pfad zu ihnen kennen.

Wenden Sie sich in Zweifelsfällen an Ihren Webhosting-Anbieter.

Jetzt können Sie anfangen. Im Folgenden finden Sie einen Überblick über die weiteren Schritte.
  1. Laden Sie die Programmdateien für den Sitemap-Generator herunter. Extrahieren Sie die Dateien in ein lokales Verzeichnis.
  2. Erstellen Sie eine Konfigurationsdatei für Ihre Website mit der zur Verfügung gestellten Datei example_config.xml als Template. Passen Sie diese Datei je nach Bedarf an Ihre Website an und speichern Sie sie.
  3. Laden Sie die erforderlichen Dateien auf Ihren Webserver herauf.
  4. Ausführen von sitemap_gen.py.
  5. Fügen Sie die generierte Sitemap Ihrem Konto für Google Webmaster-Tools hinzu.
  6. Einrichtung eines sich wiederholenden Skripts. (Optional)

Wenn es Ihnen nicht möglich ist, mit dem Sitemap-Generator zu arbeiten, können Sie das Google Webmaster-Tool auch in einem anderen Format wie einer einfachen Textdatei zu Ihrem Konto für Google Sitemaps hinzufügen. Hier finden Sie außerdem Links zu Programmen von Drittanbietern, die Google Sitemaps unterstützen.

1. Herunterladen der Sitemap-Generator-Programmdateien

Die Sitemap-Generator-Dateien sind in ZIP- und GZ-Archivformaten verfügbar, und zwar am folgenden Speicherort:

http://sourceforge.net/project/showfiles.php?group_id=137793&package_id=153422

Wenn Sie das Archiv heruntergeladen haben, extrahieren Sie es in ein lokales Verzeichnis. Lokalisieren Sie die folgenden Dateien:

  • README —enthält die neuesten Informationen über dieses Tool
  • sitemap_gen.py —das Python-Skript, mit dem Ihre Sitemap erstellt wird
  • example_config.xml —die Vorlage für die Konfigurationsdatei, die Sie verwenden, um die Konfiguration für Ihre Website anzugeben.
  • example_urllist.txt —die Vorlage für die URL-Liste, die Sie verwenden können, wenn Sie eine Sitemap anhand einer von Ihnen angegebenen URL-Liste erstellen möchten.
2a. Erstellen einer Konfigurationsdatei

In diesem Abschnitt wird Ihnen Schritt für Schritt die Erstellung einer Konfigurationsdatei gezeigt. Außerdem enthält der Abschnitt eine vollständige Referenz der verfügbaren Optionen. Wenn Sie mobile Sitemaps erstellen, beachten Sie die zusätzlichen Richtlinien für mobile Sitemaps.

Um eine Konfigurationsdatei für Ihre Website zu erstellen, müssen Sie über die folgenden Informationen verfügen:

  • Die Basis-URL für Ihre Website (wie z.B. http://www.example.com/). Stellen Sie sicher, dass Sie auch das Protokoll (wie z.B. http://) aufnehmen. Zum Beispiel: http://www.google.com ist eine gültige Basis-URL, aber www.google.com nicht.
  • Der Webserver-Pfad zum Speicherort, an dem Sie die Sitemap speichern wollen. Im Allgemeinen ist dies der Pfad zur Basis-URL, da die Sitemap keine URLs enthalten kann, die sich in einem Verzeichnis auf einer höheren Ebene im Vergleich zum Speicherort der Sitemap befinden. Wenn Sie den Google Sitemap Generator ausführen, erstellt er die Sitemap und legt Sie am von Ihnen angegebenen Speicherort ab.
  • Die Verfahren, die der Sitemap-Generator für die Erstellung Ihrer Sitmap verwenden soll. Sie können eine beliebige Kombination von Verfahren verwenden. Die folgenden Verfahren stehen zur Verfügung:
    • URL —Listen Sie individuelle URLs in diesem Abschnitt der Konfigurationsdatei auf, zusammen mit Informationen über jede URL. Dieses Verfahren wird im Allgemeinen in Kombination mit einem anderen Verfahren verwendet, um manuell zusätzliche URLs aufzunehmen, die mit anderen Verfahren nicht erfasst würden.
    • URL-Liste —verweist die Konfigurationsdatei auf eine Textdatei, die eine URL-Liste enthält. Dieses Verfahren bietet sich an, wenn diese Textdatei bereits existiert oder wenn Sie zur Erstellung einer URL-Liste ein Skript verwenden.
    • Verzeichnispfade —geben Sie die Verzeichnispfade für Ihre Website sowie die entsprechenden URLs zu diesen Pfaden an. Der Sitemap -Generator erstellt eine URL-Liste auf der Grundlage des Inhalts dieser Verzeichnisse. Dieses Verfahren bietet sich an, wenn Ihre Website aus statischen HTML-Dateien besteht.
    • Zugriffsprotokolle —verweisen auf den Pfad zu Ihren Protokolldateien. Der Sitemap-Generator erstellt eine URL-Liste auf der Grundlage der URLs in den Protokollen. Dieses Verfahren bietet sich an, wenn Ihre Website aus dynamischen Seiten besteht.
    • Sitemap — verweist auf vorhandene Sitemaps, die Sie mit dem Sitemap-Generator erstellt haben. Der Sitemap-Generator erstellt eine einzelne Sitemap, die die in jeder Sitemap enthaltenen URLs umfasst. Sie können diese Methode verwenden, wenn Sie bereits mehrere kleinere Sitemaps erstellt haben, die Sie zu einer größeren Sitemap zusammenführen möchten.

    8. URL

    Lokalisieren Sie den folgenden Abschnitt:

     
    <!-- ** MODIFY or DELETE ** 
"url" nodes specify individual URLs to include in the map. <br>

Required attributes: 
href - the URL

Optional attributes: 
lastmod - timestamp of last modification (ISO8601 format) 
changefreq - how often content at this URL is usually updated
priority - value 0.0 to 1.0 of relative importance in your site 
--> 

<url href="http://www.example.com/stats?q=name" /> 
<url 
href="http://www.example.com/stats?q=age" 
lastmod="2004-11-14T01:00:00-07:00" 
changefreq="yearly" 
priority="0.3"
/>

    In diesem Abschnitt finden Sie zwei Beispiele: Das erste enthält nur das erforderliche Attribut, das zweite sowohl das erforderliche Attribut als auch die optionalen Attribute.

    Verwenden Sie dieses Format für alle URLs, die Sie aufnehmen wollen. Das changefreq-Attributs vermittelt Google einen Eindruck davon, wie oft die URL aktualisiert wird. Dadurch erfährt Google, wie oft die Seite zur Erfassung neuer Inhalte besucht werden sollte. Das priority-Attribut informiert Google über die relative Wichtigkeit dieser Seite im Vergleich zu den anderen Seiten Ihrer Website. Dieses Attribut hat keine Auswirkung darauf, wie Google Ihre Seite im Vergleich zu Seiten anderer Websites sieht. Es informiert Google lediglich darüber, welche Seiten Ihrer Website Sie für am wichtigsten halten.

    URL-Liste

    Lokalisieren Sie den folgenden Abschnitt:

     
    <!-- ** MODIFY or DELETE **
"urllist" nodes name text files with lists of URLs. 
An example file "example_urllist.txt" is provided. 

Required attributes: 
path - path to the file 

Optional attributes: 
encoding - encoding of the file if not US-ASCII 
--> 
<urllist path="example_urllist.txt" encoding="UTF-8" />

    Mit diesem Format können Sie auf den Pfad und den Namen der Textdatei verweisen, die Ihre Liste mit den URLs enthält. Sie können die bereitgestellte Datei example_urllist.txt als Vorlage für diese Textdatei verwenden. Sie können einen relativen oder einen vollständigen Pfad zu Ihrem Webserver angeben. Wenn sich der Sitemap-Generator und die Datei urlist.txt beispielsweise im selben Verzeichnis befinden, können Sie einfach den Dateinamen der TXT-Datei angeben. Wenn Sie eine Textdatei mit einer anderen Codierung als UTF-8 erstellen, können Sie die Codierung mit dem encoding-Attribut angeben. Wenn mehrere TXT-Dateien vorhanden sind, können Sie Platzhalter verwenden. Beispiel:

    <urllist path="example_urllist*.txt" encoding="UTF-8" />

    Für jede URL, die Sie in die Textdatei aufnehmen, können Sie das Datum der letzten Veränderung angeben sowie die Häufigkeit und die Priorität ändern. Zu ausführlichen Informationen über die Struktur dieser Datei siehe den Abschnitt URL-Liste Textdatei.

    Verzeichnispfade

    Lokalisieren Sie den folgenden Abschnitt: 

    <!-- ** MODIFY or DELETE ** 
"directory" nodes tell the script to walk the file system and 
include all files and directories in the Sitemap.

Required attributes:
path - path to begin walking from 
url - URL equivalent of that path 

Optional attributes:
default_file - name of the index or default file for directory URLs

--> 
 <directory  path="/var/www/icons"    url="http://www.example.com/images/" />
    <directory
    path="/var/www/docroot"
    url="http://www.example.com/"
    default_file="index.html"
    />

    In diesem Abschnitt werden zwei Beispiele gezeigt. Wenn Ihre sämtlichen Seiten in den Unterverzeichnissen eines Pfades enthalten sind, brauchen Sie nur einen Eintrag aufzunehmen. Wenn Sie jedoch mehrere Pfade zu Seiten auf Ihrer Website haben, müssen Sie für jeden einen Eintrag aufnehmen.

    Beachten Sie, dass jede URL mit der von Ihnen in Schritt 3 angegebenen Basis-URL beginnen muss. Zum Beispiel: Die in der example_config.xml-Datei enthaltenen Beispiele weisen beide URLs auf, die mit http://www.example.com/ beginnen. Deshalb sind beide URLs gültig.

    Ersetzen Sie die Beispiel-Einträge mit Einträgen für Ihre Website. Viele Websites haben nur einen Eintrag, der auf die Basis-URL verweist. Stellen Sie sicher, dass der path-Wert der vollständige Pfad zum Verzeichnis auf Ihrem Webserver ist. Stellen Sie sicher, dass der url-Wert die vollständige URL ist, einschließlich des Protokolls (wie z.B. http) und erforderlichenfalls eines abschließenden Schrägstrichs.

    Sie können die default_file-Parameter verwenden, um den Dateinamen anzugeben, den Ihr Server als Default-Seite für ein Verzeichnis verwendet. Im oben aufgeführten Beispiel wird /var/www/docroot in http://www.example.com/index.html übersetzt. Sie brauchen dies nicht anzugeben. Wenn Sie dies jedoch tun, schließt der Sitemap-Generator die jedem Unterverzeichnis entsprechende Seite nur einmal ein (anstatt sowohl die Verzeichnis-URL als auch die Dateinamen-URL anzugeben) und verwendet das Datum der letzten Veränderung der Datei (anstatt des Verzeichnisses) zum Extrahieren des lastmod-Attributs für diese Seite.

    Zugriffsprotokolle

    Lokalisieren Sie den folgenden Abschnitt: 

    <!-- ** MODIFY or DELETE **
"accesslog" nodes tell the script to scan webserver log files to
extract URLs on your site.  Both Common Logfile Format (Apache's default 
logfile) and Extended Logfile Format (IIS's default logfile) can be read.
				
Required attributes:
 path - path to the file
Optional attributes:
 encoding - encoding of the file if not US-ASCII
  -->
<accesslog path="/etc/httpd/logs/access.log" encoding="UTF-8" />
<accesslog path="/etc/httpd/logs/access.log.0" encoding="UTF-8" />
<accesslog path="/etc/httpd/logs/access.log.1.gz" encoding="UTF-8" />

    Dieser Abschnitt enthält drei Beispiele. Sie sollten diese Einträge durch einen Eintrag für jede Protokolldatei ersetzen. Stellen Sie sicher, dass als Wert für "path" der vollständige Pfad- und Dateiname auf Ihrem Webserver angegeben wird. Wenn die Protokolldateien nicht als US-ASCII oder UTF8 codiert sind, verwenden Sie das optionale Attribut encoding, um die Codierung anzugeben. Darüber hinaus können Platzhalter angegeben werden, anstatt jede Protokolldatei einzeln aufzuführen. Im obigen Beispiel könnten Sie beispielsweise den folgenden Eintrag angeben, damit alle drei Protokolldateien eingeschlossen werden:

    <accesslog path="/etc/httpd/logs/access.log*" encoding="UTF-8" />

    Der Sitemap-Generator ordnet den in den Protokollen gefundenen URLs eine Priorität zu entsprechend der Häufigkeit, mit der auf eine URL zugegriffen wird. Zum Beispiel: Eine URL, auf die 100 Mal zugegriffen wurde, erhält eine höhere Priorität als eine URL, auf die zweimal zugegriffen wurde. Die Prioritätszuordnung ist relativ und hängt von jeder URL in Relation zu anderen URLs auf der Website ab.

    sitemap

    Lokalisieren Sie den folgenden Abschnitt: 

    <!-- ** MODIFY or DELETE **
    "sitemap" nodes tell the script to scan other Sitemap files. This can
be useful to aggregate the results of multiple runs of this script into
a single Sitemap.

Required attributes:
path - path to the file
-->
<sitemap path="/var/www/docroot/subpath/sitemap.xml" />

    Dieser Abschnitt enthält ein Beispiel. Sie sollten diesen Eintrag durch einen Eintrag für jede Sitemap ersetzen, die Sie einschließen möchten. Stellen Sie sicher, dass als Wert für "path" der vollständige Pfad- und Dateiname auf Ihrem Webserver angegeben wird. Sie können auch mit gzip komprimierte Sitemaps aufführen, sofern diese die Dateinamenerweiterung .gz aufweisen. Darüber hinaus können Platzhalter angegeben werden, anstatt jede Sitemap einzeln aufzuführen. Mit dem folgenden Eintrag wird beispielsweise jede Sitemap eingeschlossen, die mit dem Wort "sitemap" beginnt und die Dateinamenerweiterung .xml aufweist:

    <sitemap path="/var/www/docroot/subpath/sitemap*.xml" />

    Der Sitemap-Generator extrahiert alle URLs sowie die für jede URL aufgeführten optionalen Daten für jede Sitemap, die Sie aufführen, und erstellt eine Sitemap mit diesen Informationen. Wir können derzeit jedoch nicht garantieren, dass mit dieser Methode Sitemaps verarbeitet werden können, die mit anderen Tools als dem Sitemap-Generator erstellt wurden.

  8. Lokalisieren Sie den Abschnitt Filterdefinition:
    9.  
    <!-- ********************************************************         
FILTERS
				
Filters specify wild-card patterns that the script compares
against all URLs it finds. Filters can be used to exclude
certain URLs from your Sitemap, for instance if you have
hidden content that you hope the search engines don't find.

Filters can be either type="wildcard", which means standard
path wildcards (* and ?) are used to compare against URLs,
 or type="regexp", which means regular expressions are used
to compare.

Filters are applied in the order specified in this file.
An action="drop" filter causes exclusion of matching URLs.
An action="pass" filter causes inclusion of matching URLs,
shortcutting any other later filters that might also match.
If no filter at all matches a URL, the URL will be included.
Together you can build up fairly complex rules.

The default action is "drop".
The default type is "wildcard".

You can MODIFY or DELETE these entries as appropriate for
your site. However, unlike above, the example entries in
this section are not contrived and may be useful to you as
they are.
********************************************************* -->

<!-- Exclude URLs that end with a '~' (IE: emacs backup files) -->
<filter action="drop" type="wildcard" pattern="*~" />

<!-- Exclude URLs within UNIX-style hidden files or directories -->
<filter action="drop" type="regexp" pattern="/\.[^/]*" />

    Sie können durch Filtern bestimmte URLs von der erstellten Sitemap ausschließen. Dies bietet sich an, wenn Sie eine sauberere Liste erstellen möchten, redundante Einträge reduzieren möchten oder verhindern möchten, dass bestimmte URLs indiziert werden. Bitte beachten Sie, dass, wenn Sie eine robots.txt-Datei verwenden, um URLs daran zu hindern, in den Index aufgenommen zu werden, Google die URLs auch dann nicht durchsucht oder indiziert, wenn die URLs in Ihrer Sitemap aufgenommen sind.

    Sie können eine oder alle der Filtermethoden verwenden. Sie können die Einträge, die Sie nicht benötigen, löschen und gegebenenfalls zusätzliche Einträge erstellen. Unten finden Sie Beispielanwendungen. 

    <filter action="drop" type="wildcard" pattern="*.jpg" />

    Dieser Filter schließt URLs aus, die auf .jpg enden. Es bietet sich an, einen ähnlichen Filter aufzunehmen, wenn alle Abbildungen auf Ihrer Website in HTML-Seiten integriert sind und nicht über eigenständige URLs auf sie zugegriffen werden sollte. 

    <filter action="pass" type="wildcard" pattern="*.htm*" />
<filter action="drop" type="wildcard" pattern="*" />
    Dieser Filter schließt alle .htm*-Dateien ein und alles Weitere aus.

Config File Syntax Reference

Eine vollständige Erklärung der Syntax der Konfigurationsdatei finden Sie unten. Jedes Tag beginnt mit einem Codebeispiel, gefolgt von einer Beschreibung der Attribute.

site
Erforderlicher Tag am Anfang jeder Konfigurationsdatei. 

<site
base_url="http://www.example.com/"
store_into="/var/www/html/sitemap.xml.gz"
verbose="1"
supress_search_engine_notify="1"
default_encoding="UTF-8">

base_url
 erforderlich Der HTTP-Pfad zur Basis Ihrer Website - nur URLs, die mit dieser Basisadresse beginnen, können in die Sitemap aufgenommen werden 
store_into
 erforderlich Der Webserver-Pfad zu der gewünschten Ausgabedatei. Diese Datei wird vom Skript erstellt - es ist nicht notwendig, die Datei vor dem Ausführen des Skripts zu erstellen. 
verbose
 Optional Geben Sie eine Zahl von 0 bis 3 ein (höhere Zahlen entsprechen umfangreicheren Debugging-Informationen) entsprechen 
suppress_search_engine_notify
 Optional Geben Sie eine "1" ein, um die Benachrichtigung der Suchmaschine zu Testzwecken zu deaktivieren. 
default_encoding
 Optional Geben Sie eine Zeichencodierung an, die auf die Dateisystempfade und URLs angewendet werden soll

url
Optionales Tag, das Sie verwenden können, um jede URL auf Ihrer Website aufzulisten. 

<url
href="http://www.example.com/stats?q=age" 
lastmod="2004-11-14T01:00:00-07:00" 
changefreq="yearly" 
priority="0.3"
/>

href
 erforderlich Der HTTP-Pfad zur Basis Ihrer Website - nur URLs, die mit dieser Basisadresse beginnen, können in die Sitemap aufgenommen werden 
lastmod
 Optional Der Zeitpunkt, zu dem die URL zuletzt geändert wurde, im Format W3C Datetime (JJJJ-MM-TT hh:mm:ss+00:00). Die Zeitangabe können Sie weglassen. Beispiele:
  "2005-02-21T18:00:15+00:00"
  "2005-02-21"		 
changefreq
 Optional Die Häufigkeit, mit der die URL sich wahrscheinlich ändert. Diese Angabe ist als Hinweis zu betrachten, nicht als Befehl. Als Wert muss "always", "hourly", "daily", "weekly", "monthly", "yearly" oder "never" angegeben werden. 
priority
 Optional Die Priorität dieser Seite im Vergleich zu anderen Seiten derselben Website. Der Wert ist eine Zahl zwischen 0.0 und 1.0, wobei 0.0 der niedrigsten und 1.0 der höchsten Priorität entspricht. Die Priorität kann sich auf die Reihenfolge auswirken, in der die URLs Ihrer Website von den Suchmaschinen zur Durchsuchung ausgewählt werden. Da die Priorität relativ ist, wird sie nur für die Auswahl zwischen URLs auf Ihrer eigenen Website verwendet; die Priorität Ihrer Seiten wird nicht mit der Priorität von Seiten anderer Websites verglichen.

urllist
Optionales Tag, das Sie dazu verwenden können, auf eine Textdatei mit einer Liste von auf Ihrer Website enthaltenen URLs hinzuweisen. 

<urllist path="/var/www/html/urllist.txt" encoding="UTF-8" />

path
 erforderlich Pfad und Dateiname der TXT-Datei. Sie können einen relativen oder einen vollständigen Pfad angeben. 
encoding
 Optional Die Codierung der Datei, sofern nicht UTF-8.

Die urllist.txt-Datei ist eine einfache Textdatei, die eine zu verarbeitende URL-Liste enthält. Sie können außerdem für jede URL optionale Attribute aufnehmen. Attribute werden in derselben Zeile wie die URL eingegeben und durch ein einfaches Leerzeichen getrennt. Zum Beispiel:

http://www.example.com/abc/something
http://www.example.com/abc/xyy.pdf lastmod=2001-12-31T14:05:06+00:00
http://www.example.com/abc/def?x=12&y=23 changefreq=weekly priority=0.3

lastmod
 Optional

Der Zeitpunkt, zu dem die URL zuletzt geändert wurde, im Format W3C Datetime (JJJJ-MM-TT hh:mm:ss+00:00). Die Zeitangabe können Sie weglassen. Beispiele:

  "2005-02-21T18:00:15+00:00"
  "2005-02-21"

 
changefreq
 Optional Die Häufigkeit, mit der die URL sich wahrscheinlich ändert. Dies wird als Hinweis und nicht als Befehl betrachtet. Der Wert muss entweder "always", "hourly", "daily", "weekly", "monthly", "yearly", oder "never" sein. 
priority
 Optional Die Priorität dieser Seite im Vergleich zu anderen Seiten derselben Website. Der Wert ist eine Zahl zwischen 0.0 und 1.0, wobei 0.0 der niedrigsten und 1.0 der höchsten Priorität entspricht. Die Priorität kann sich auf die Reihenfolge auswirken, in der die URLs Ihrer Website von den Suchmaschinen zur Durchsuchung ausgewählt werden. Da die Priorität relativ ist, wird sie nur für die Auswahl zwischen URLs auf Ihrer eigenen Website verwendet; die Priorität Ihrer Seiten wird nicht mit der Priorität von Seiten anderer Websites verglichen.

directory
Optionales Tag, das Sie verwenden können, um Verzeichnisse auf Ihrer Website näher zu bezeichnen, sodass der Sitemap-Generator eine Liste von URLs von den in diesen Verzeichnissen gefundenen Dateien erstellen kann. 

--> 
 <directory  path="/var/www/icons"    url="http://www.example.com/images/" />
<directory
path="/var/www/docroot"
url="http://www.example.com/"
default_file="index.html"
/>
path
 erforderlich Gibt den Anfangspfad an. Der Sitemap-Generator durchsucht dieses Verzeichnis sowie alle Unterverzeichnisse. 
url
 erforderlich Gibt das URL-Äquivalent des path-Werts an. 
default_file
 Optional Gibt die standardmäßige Datei für ein Verzeichnis auf dem Server an.

accesslog
Optionales Tag, das Sie verwenden können, um den Pfad und Dateinamen von Zugriffsprotokolldateien im IIS- und Apache-Stil näher zu bezeichnen, sodass der Sitemap-Generator automatisch URLs von ihnen abholen kann. 

<accesslog path="/etc/httpd/logs/access-0.log" encoding="UTF-8"/>
path
 erforderlich Gibt den Pfad zur Datei an. 
encoding
 Optional Gibt die Codierung der Datei näher an, wenn sie eine andere ist als UTF-8.

sitemap
Optionales Tag, das zur Angabe des Pfades und des Dateinamens von vorhandenen Sitemaps verwendet werden kann, die Sie mit dem Sitemap-Generator erstellt haben. Der Sitemap-Generator erstellt eine einzelne Sitemap, die die in jeder Sitemap enthaltenen URLs umfasst. 

<Sitemap Pfad="/var/www/docroot/subpath/sitemap.xml" />
path
 erforderlich Gibt den Pfad zur Sitemapdatei an.

filter
Optionales Tag, das Sie zur Erstellung von Regeln verwenden können, mit denen bestimmte Dateien ein- oder ausgeschlossen werden. Filter werden in der Reihenfolge angewendet, in der sie in der config.xml-Datei aufgeführt sind. Das Vermischen von Filtereinträgen mit Eingabeeinträgen (url, urllist, directory oder accesslog) hat jedoch keine weiteren Auswirkungen - jede vom Sitemap-Generator zur Sitemap hingezufügte URL wird zunächst gegen jeden Filter abgeglichen. Wenn eine URL mit keinem Filter übereinstimmt, wird sie gemäß der Standardeinstellung in die Sitemap aufgenommen.

<filter action="drop" type="wildcard" pattern="*/internal/*" />
action
 Optional

Die vom Filter durchzuführende Aktion. Gültige Werte sind:
  • drop - schließt übereinstimmende URLs aus. Dieser Vorgang ist voreingestellt; wenn also kein Vorgang angegeben ist, geht der Generator von der Funktion "drop" aus.
  • pass - schließt übereinstimmende URLs ein.

 
type
 Optional

Art der Filterung. Gültige Werte sind:
  • wildcard - beim URL-Abgleich werden standardmäßige Pfadplatzhalterzeichen (? und *) verwendet. Dies ist der voreingestellte Typ; wenn also kein Typ angegeben wird, geht der Generator von "wildcard".
  • regexp - reguläre Ausdrücke werden verwendet.
 
pattern
 erforderlich Gibt das Muster für den Abgleich an.

Codierungen

Die in Ihrer Konfigurationsdatei aufgeführten Dateien, seien es URL-Listen oder Webserver-Protokolle, können andere Codierungen als das standardmäßige UTF-8 verwenden. Sie können alternative Codierungen in der Datei config.xml angeben, um zu beeinflussen, wie die Dateien vom Sitemap-Generator gelesen werden. Folgende Codierungen werden häufig verwendet:

  • encoding="utf-8" ist die angenommene Standardeinstellung
  • encoding="ascii" ist eine Untermenge von UTF-8, braucht daher nicht angegeben zu werden
  • encoding="iso-8859-1" ist für viele westeuropäische Sprachen üblich
2b. Zusatzinformationen zum Erstellen einer mobilen Konfigurationsdatei

Konfigurationsdateien für mobile Sitemaps werden auf dieselbe Weise erstellt wie für nicht mobile Sitemaps. Jedoch muss für jede Auszeichnungssprache eine eigene Konfigurationsdatei erstellt werden, und der Sitemap-Generator muss mit jeder Konfigurationsdatei einzeln ausgeführt werden, so dass eine separate Sitemap für jede Sprache erzeugt wird.

Jede Konfigurationsdatei muss:

  • Geben Sie einen anderen Dateinamen für den Wert store_into an.
  • Verwenden Sie Filter, um die URLs anzugeben, die für die betreffende Auszeichnungssprache aus- bzw. eingeschlossen werden sollen. Beachten Sie, dass jede Sitemap nur URLs für eine einzelne Auszeichnungssprache enthalten darf. Eine URL kann daher in mehrere Sitemaps aufgenommen werden, wenn sie mehrere Auszeichnungssprachen liefert.

Beispiele für die Filterung

Nachstehend sehen Sie einige Beispiele dafür, wie Sie mit Hilfe erweiterungsbasierter Filter mobile Sitemaps für verschiedene Auszeichnungssprachen generieren können. Welche Filterung verwendet werden sollte, hängt davon ab, welche Typen von Auszeichnungssprachen auf Ihrer Website verwendet werden und wie die einzelnen Typen angegeben wurden. Wenn Einzelheiten Ihrer Website anders implementiert sind (beispielsweise wenn Sie URLs für verschiedene Auszeichnungssprachen in separaten Ordnern organisiert haben), müssen die Filter gemäß den spezifischen Merkmalen Ihrer Websiteimplementierung angewendet werden. Beachten Sie, dass Filter in der Reihenfolge angewendet werden, in der sie in der Konfigurationsdatei aufgeführt sind. Als ersten Filter sollten Sie daher eine "pass"-Aktion einrichten, die die URLs enthält, die in die Sitemap aufgenommen werden sollen.

So erstellen Sie eine Sitemap für WML-Content (WAP 1.2):

<filter action="pass" type="wildcard" pattern="*.wml" />
<filter action="drop" type="wildcard" pattern="*.*" />

So erstellen Sie eine Sitemap für XHTML Mobile Profile-Content (WAP 2.0):

<filter action="pass" type="wildcard" pattern="*.xhtml" />
<filter action="drop" type="wildcard" pattern="*.*" />
3. Hochladen der Dateien auf Ihren Webserver

Sie sollten die folgenden Dateien auf Ihren Webserver in einen Speicherort hochladen, auf den Sie von einer Befehlszeile aus zugreifen können:

  • config.xml —Dies ist die Konfigurationsdatei, die Sie gerade mit example_config.xml erstellt haben.
  • sitemap_gen.py —Dies ist das Python-Skript, mit dem Ihre Sitemap erstellt wird.
  • urllist.txt —Diese Datei ist optional; Sie brauchen Sie nur dann aufzunehmen, wenn Sie eine Sitemap mit dem Textdateiverfahren erstellt haben.

Das von Ihnen zum Hochladen dieser Dateien verwendete Verfahren hängt von der Umgebung ab. Gängige Verfahren sind FTP und SCP. Für weitere Informationen können Sie sich an Ihren Webhosting-Anbieter wenden.

Für weitere Informationen über das Herstellen einer Verbindung zu Ihrem Webserver und über das Ausführen von Skripts sollten Sie sich an Ihren Webhosting-Anbieter wenden.

5. Einreichen Ihrer Sitemap bei Google

Der Sitemap-Generator erstellt die Datei sitemap.xml.gz an dem Speicherort, den Sie in der Konfigurationsdatei angegeben haben. Wenn diese Datei erfolgreich erstellt wurde, stellen Sie sicher, dass der Zugriff darauf über einen Webbrowser erfolgen kann. Fügen Sie sie anschließend Ihrem Konto für Google Sitemaps hinzu. Nur so kann Google Ihnen nützliche Status- und Statistikinformationen zur Verfügung stellen. Wenn Google Probleme mit Ihrer Sitemap meldet, können Sie die Probleme beheben und die Sitemap erneut einreichen. Sie müssen die Sitemap nur einmal manuell hinzufügen. Danach können Sie Google mithilfe einer HTTP-Anforderung über Änderungen an Ihrer Sitemap in Kenntnis setzen. (Sie können sie aber auch über Ihr Konto für Webmaster-Tools von Google erneut einreichen).

6. Einrichtung eines sich wiederholendes Skripts

Es ist zu empfehlen, bei jeder Änderung des Inhalts die Ausführung des Sitemap-Generators zu veranlassen, bis zu einer maximalen Frequenz von einmal pro Stunde.

Bei einem UNIX-Webserver kann der Webmaster überlegen, diesen Vorgang als cron-Job zu konfigurieren.

Webmaster, die andere Plattformen nutzen, sollten sich bei der Konfiguration sich wiederholender Skripts von Ihrem Systemadministrator helfen lassen. Sie können auch Ratschläge von anderen Nutzern in der Google Sitemaps-Gruppe in Google Groups einholen.

Mithilfe einer HTTP-Anforderung können Sie Google über Änderungen an Ihrer Sitemap in Kenntnis setzen. Melden Sie sich einmal mit Ihrem Google-Konto in Google Webmaster-Tools an, um Ihre Sitemap manuell zu Ihrem Konto für Google Webmaster-Tools hinzuzufügen.

Fehlersuche

In diesem Abschnitt nennen wir einige häufig gestellte Fragen sowie Probleme, die manche Nutzer beim Bedienen des Sitemap-Generator haben, und weisen auf mögliche Antworten und Lösungen hin.

Über das Web zugänglich

Issue: Der Sitemap-Generator gibt die folgende Fehler- und Warnmeldung zurück:

 
[ERROR] When attempting to access your generated Sitemap at the following URL:
	http://www.example.com/sitemap.xml.gz
	we failed to read it.  Please verify the store_into path you specified in
	your configuration file is web-accessible.  Consult the FAQ for more
	information.
	[WARNING] Proceeding to notify with an unverifiable URL.

Was ist passiert? Der Sitemap-Generator hat am von Ihnen angegebenen Ort eine Datei erstellt und hat dann - wie eine Suchmaschine - versucht, sie mit HTTP abzurufen, jedoch ohne Erfolg. Er hat trotzdem die Suchmaschinen informiert, gibt aber eine Fehlermeldung aus, um Ihnen mitzuteilen, dass Ihre Sitemap möglicherweise von den Suchmaschinen nicht gelesen werden kann.

Der Sitemap-Generator erstellt Ihre Sitemap-Datei in dem Pfad, der im store_into-Attribut in Ihrer config.xml-Datei angegeben ist. Anschließend erstellt der Sitemap-Generator anhand des base_url-Attributs eine URL für diese Datei und teilt die URL den Suchmaschinen mit. Wenn Sie z.B. in der Konfiguration folgende Werte angeben:

 
<site base_url="http://www.example.com/"
	      store_into="/var/www/html/sitemap.xml.gz">

Der Sitemap-Generator teilt Suchmaschinen den folgenden Speicherort für ihre Sitemap mit:

http://www.example.com/sitemap.xml.gz

Auf die Datei muss über diese URL zugegriffen werden können. Wenn die Datei nicht über diese URL abgerufen werden kann, können Suchmaschinen Ihre Sitemap nicht finden.

Was können Sie tun? Sie müssen sicherstellen, dass Ihre config.xml-Datei die richtigen base_url- und store_into-Pfade für Ihre Sitemap angibt. Sie müssen außerdem überprüfen, dass Webbrowser die Sitemap-Datei von der base_url abrufen können. Wenn Sie im Pfad oder in der URL Fehler finden, müssen Sie vielleicht den Sitemap-Generator noch einmal aktivieren, oder Sie bringen die Sitemap-Datei zum richtigen Speicherort und informieren die Suchmaschinen manuell über ihre Websites.

Unterstützung für XML

Problem: Der Sitemap-Generator gibt die folgende Fehlermeldung aus:

 
Some installs of Python 2.2 do not include complete support for XML. 
	Please try upgrading your version of Python and re-running the script.

Was ist passiert? Wie die Meldung besagt, wird auf einigen Plattformen eine Version von Python (Sprache, in der das Sitemap-Generator-Skript verfasst ist) verwendet, der die für die Verarbeitung von XML-Dateien erforderlichen Supportbibliotheken fehlen. Das Skript benötigt zur Ausführung jedoch die vollständige XML-Unterstützung.

Was können Sie tun? Versuchen Sie, Ihre installierte Python-Version durch eine neuere Version zu ersetzen. Dazu müssen Sie sich wahrscheinlich an Ihren Systemadministrator wenden.

Bitte beachten Sie, dass sich dies nur auf den Sitemap-Generator auswirkt und nicht auf Sitemaps allgemein. Wenn Sie ein anderes Verfahren oder Tool für die Erstellung von Sitemaps haben, können Sie dies durchaus verwenden und Ihre Sitemaps bei Suchmaschinen einreichen.

Was sind die zusätzlichen Dateien in der .gz- oder .zip-Datei?

Wenn Sie das Skript sitemap_gen.py extrahieren, sehen Sie wahrscheinlich deutlich mehr Dateien als nur die oben genannten. Die vollständige Dateiliste sieht etwa so aus:

 
AUTHORS
	ChangeLog
	COPYING
	example_config.xml
	example_urllist.txt
	PKG-INFO
	README
	setup.py
	sitemap_gen.py
	test_sitemap_gen.py

Die zusätzlichen Dateien enthalten z.B. Informationen zum Softwarepaket und Lizenzbedingungen. Wir empfehlen Ihnen, diese Dateien zu lesen.

Die einzige Ausnahme ist test_sitemap_gen.py, ein Unit-Testskript, das für Entwickler, die einen Beitrag zu diesem Open Source-Projekt leisten möchten, nützlich sein kann. Wenn Sie den Sitemap-Generator verwenden möchten, ohne den Quellcode zu verändern - was für fast alle Nutzer gilt -, brauchen Sie dieses Testskript wahrscheinlich nicht.

Wenn Sie sich an diesem Projekt beteiligen möchten, besuchen Sie die Webseite http://sourceforge.net/projects/goog-sitemapgen. Hier finden Sie zudem auch Links zu Drittanbieterprogrammen, die Google Sitemaps unterstützen.

Nicht-ASCII-Zeichen

Problem: Der Domänenname Ihrer Website oder die hierin enthaltenen URLs enthalten Nicht-ASCII-Zeichen.

Im Allgemeinen sollten URLs, die nicht aus ASCII-Zeichen bestehen, mit UTF-8 codiert werden, bevor sie mit Prozent (%) als Escapezeichen versehen werden. Allerdings antworten einige Webserver nur dann ordnungsgemäß, wenn URLs mit einer anderen Codierung als UTF-8 codiert werden. Alle URLs in Ihrer Sitemap sowie die URL der Sitemap selbst müssen so codiert werden, dass sie von dem Webserver, auf dem sie sich befinden, gelesen werden können. Verwenden Sie im siteDefinitionsabschnitt das optionale Attribut default_encoding, um die von Ihrem Webserver verwendete Codierung anzugeben. Wenn Sie dieses Attribut nicht angeben und Ihr Webserver eine andere Codierung als UTF-8 verwendet, wissen wir nicht, welche Codierung verwendet werden soll.

Wenn Ihre URLs Nicht-ASCII-Zeichen umfassen, empfehlen wir, das Skript des Sitemap-Generators unter Verwendung von Python 2.3 oder höher auszuführen. Diese Version von Python wurde um die Unterstützung von Nicht-ASCII-Zeichen erweitert. Wenn Ihre Domänenamen Nicht-ASCII-Zeichen umfassen, müssen Sie Python 2.3 oder höher verwenden, da die Unterstützung für Internationalizing Domain Names in Applications (IDNA) erst mit dieser Version implementiert wurde. Ohne IDNA-Unterstützung kann der Sitemap-Generator Domänennamen, die Nicht-ASCII-Zeichen umfassen, nicht ordnungsgemäß codieren.
