Înainte de a începe

Generatorul de sitemap-uri Google este un script Python care creează un sitemap pentru site-ul dvs. utilizând Protocolul pentru sitemap-uri. Scriptul poate crea sitemap-uri din liste de adrese URL, directoare de servere de Web sau din fişiere jurnal de acces. Pentru a utiliza acest script:

  • Trebuie să vă puteţi conecta la, şi să rulaţi scripturi pe, serverul dvs. de Web.
  • Serverul dvs. de Web trebuie să aibă instalat Python 2.2 sau o versiune mai recentă.
  • Trebuie să ştiţi comanda care lansează Python. (În general, aceasta este python, dar poate varia în funcţie de instalare. De exemplu, dacă serverul de Web are două versiuni de Python instalate, versiunea iniţială poate fi invocată prin comanda python iar versiunea ulterioară poate fi invocată prin comanda python2.)
  • Trebuie să cunoaşteţi calea site-ului dvs. Dacă serverul dvs. de Web găzduieşte un singur site, această cale poate fi de felul var/www/html. Dacă aveţi un server virtual care găzduieşte mai multe site-uri, această cale poate fi de felul home/virtual/site1/fst/var/www/html.
  • Trebuie să aveţi posibilitatea de a încărca fişierele pe serverul dvs. de Web (de exemplu, utilizând FTP).
  • Dacă veţi genera o listă de adrese URL pe baza fişierelor jurnal de acces, trebuie să cunoaşteţi codificarea utilizată pentru acele fişiere jurnal şi calea completă către ele.

Dacă nu sunteţi sigur de niciuna dintre acestea, puteţi verifica prin compania dvs. de găzduire Web.

Acum sunteţi pregătit să începeţi. Aici găsiţi o prezentare generală a ceea ce trebuie să faceţi.

  1. Descărcaţi fişierele de program ale Generatorului de sitemap-uri. Extrageţi fişierele într-un director local.
  2. Creaţi un fişier de configuraţie pentru site-ul dvs., utilizând ca şablon fişierul exemplu_config.xml furnizat. Modificaţi acest fişier aşa cum este necesar pentru site-ul dvs. şi salvaţi-l.
  3. Încărcaţi fişierele necesare pe serverul dvs. de Web.
  4. Executaţi sitemap_gen.py.
  5. Adăugaţi sitemap-ul generat în contul dvs. de instrumente Google pentru webmasteri.
  6. Setarea unui script recurent. (opţional)

Dacă nu puteţi utiliza Generatorul de sitemap-uri, puteţi adăuga un sitemap în contul dvs. de instrumente Google pentru webmasteri în alt format, cum ar fi un fişier text simplu. De asemenea, puteţi găsi link-uri către programe terţe care acceptă sitemap-uri Google aici.

Pentru sitemap-urile de ştiri: Generatorul de sitemap-uri nu este recomandat momentan pentru utilizare în crearea sitemap-urilor pentru Ştiri Google din cauza cerinţelor speciale ale sitemap-urilor de ştiri. Sitemap-urile de ştiri sunt destinate să fie liste dinamice numai cu articolele de ştiri cel mai recent publicate (nu pentru întregul site de Web) şi sunt actualizate frecvent.

1. Descărcarea fişierelor de program pentru Generatorul de sitemap-uri

Fişierele Generatorului de sitemap-uri sunt disponibile în formatele de arhivare ZIP sau GZ din următoarea locaţie:

http://sourceforge.net/project/showfiles.php?group_id=137793&package_id=153422

După ce descărcaţi arhiva, extrageţi-o într-un director local. Localizaţi următoarele fişiere:

  • README —conţine cele mai recente informaţii despre acest instrument
  • sitemap_gen.py —script-ul python care generează sitemap-ul dvs.
  • exemplu_config.xml —fişierul de configurare şablon pe care îl veţi utiliza pentru a specifica configuraţia pentru site-ul dvs.
  • exemplu_listaurl.txt —lista şablon de adrese URL pe care o puteţi utiliza dacă doriţi să creaţi un sitemap bazat pe un set de adrese URL specificate de dvs.
2a. Crearea unui fişier de configuraţie

Această secţiune furnizează instrucţiuni pas cu pas pentru crearea unui fişier de configuraţie. De asemenea, furnizează referinţe complete pentru opţiunile disponibile. Când creaţi sitemap-uri mobile, verificaţi indicaţiile suplimentare pentru mobile.

Pentru a crea un fişier de configuraţie pentru site-ul dvs., trebuie să aveţi următoarele informaţii:

  • Adresa URL de bază pentru site-ul dvs. (cum ar fi http://www.exemplu.ro/). Asiguraţi-vă că aţi inclus protocolul (cum ar fi http://). De exemplu, http://www.google.ro este o adresă URL de bază valabilă, însă www.google.ro nu este.
  • Calea serverului de Web către locaţia unde doriţi să stocaţi sitemap-ul. În general, aceasta este calea către adresa URL de bază, deoarece sitemap-ul nu poate conţine adrese URL care se află într-un director de nivel mai înalt decât locaţia sitemap-ului. Când executaţi Generatorul de sitemap-uri Google, acesta creează sitemap-ul şi îl plasează în locaţia specificată de dvs.
  • Metodele pe care doriţi să le utilizeze Generatorul de sitemap-uri pentru a crea sitemap-ul dvs. Puteţi utiliza orice combinaţie de metode. Sunt disponibile următoarele metode:
    • Adresa URL—listaţi adresele URL individuale din această secţiune a fişierului de configuraţie, împreună cu informaţii despre fiecare adresă URL. Această metodă se utilizează în general împreună cu o altă metodă, pentru a include manual adrese URL suplimentare, pe care alte metode nu le-ar alege.
    • Listă de adrese URL —direcţionaţi fişierul de configuraţie către un fişier text care conţine o listă de adrese URL. Puteţi utiliza această metodă dacă acest fişier text există deja sau dacă utilizaţi un script pentru a genera o listă de adrese URL.
    • Căi de directoare—specificaţi căile pentru site-ul dvs. şi adresele URL corespunzătoare acelor căi. Generatorul de sitemap-uri va crea o listă de adrese URL pe baza conţinutului acelor directoare. E posibil să doriţi să utilizaţi această metodă dacă site-ul dvs. constă din fişiere HTML statice.
    • Fişierele jurnal de acces—precizaţi calea către fişierele dvs. jurnal. Generatorul de sitemap-uri va crea o listă de adrese URL pe baza adreselor URL incluse în fişierele jurnal. Puteţi utiliza această metodă dacă site-ul dvs. constă din pagini dinamice.
    • Sitemap—indicaţi sitemap-urile existente pe care le-aţi creat cu Generatorul de sitemap-uri. Generatorul de sitemap-uri va crea un singur sitemap care va conţine toate adresele URL incluse în fiecare sitemap. Aţi putea utiliza această metodă dacă aţi creat deja câteva sitemap-uri mai mici pe care doriţi să le uniţi într-un sitemap mai mare.

Creaţi fişierul de configuraţie după cum urmează:
  1. Deschideţi fişierul exemplu_config.xml într-un editor de text. Salvaţi-l ca fişier nou (cum ar fi config.xml sau site-ulmeu_config.xml).

  2. Localizaţi secţiunea de definire a site-ului: 
    3. <site base_url="http://www.exemplu.ro/" store_into="/var/www/docroot/sitemap.xml.gz"verbose="1">
  3. Schimbaţi valoarea base_url cu adresa URL a site-ului dvs.
  4. Schimbaţi valoarea store_into cu calea de pe serverul dvs. de Web unde doriţi să stocaţi sitemap-ul şi numele de fişier pe care doriţi să îl utilizaţi pentru sitemap. În general, aceasta este calea către adresa URL de bază deoarece Google poate să accepte numai adrese URL care sunt la acelaşi nivel al, sau subdirectoare ale, directorului care conţine sitemap-ul. Puteţi specifica o cale relativă de la directorul unde aţi încărcat script-ul sau o cale completă din rădăcina serverului dvs. de Web. Dacă încărcaţi script-ul în directorul adresei URL de bază, puteţi specifica doar numele fişierului.

  5. Localizaţi secţiunile metodei de generare care încep cu ** MODIFICAŢI sau ŞTERGEŢI **. Fiecare dintre aceste secţiuni corespunde unei metode de generare a unui sitemap.
  6. Ştergeţi secţiunile pentru metodele pe care nu le veţi utiliza.

    8. Adresa URL

    Localizaţi următoarea secţiune: 

    <!-- ** MODIFY or DELETE ** 
"url" nodes specify individual URLs to include in the map. <br>

Required attributes: 
href - the URL

Optional attributes: 
lastmod - timestamp of last modification (ISO8601 format) changefreq - how often content at this URL is usually updated priority - value 0.0 to 1.0 of relative importance in your site --> 

<url href="http://www.exemplu.ro/statistici?q=nume" />
<url href="http://www.exemplu.ro/statistici?q=vârsta" 
lastmod="2004-11-14T01:00:00-07:00" 
changefreq="yearly" 
priority="0.3"
/>

    Această secţiune oferă două exemple: primul include doar atributul obligatoriu, iar al doilea include atât atributul obligatoriu, cât şi atributele opţionale.

    Utilizaţi acest format pentru fiecare dintre adresele URL pe care doriţi să le includeţi. Atributul changefreq oferă pentru Google o idee generală despre frecvenţa cu care este actualizată adresa URL. Aceasta ajută Google să afle cât de des trebuie să viziteze pagina pentru un conţinut nou. Atributul priority oferă pentru Google informaţii despre importanţa relativă a acestei pagini în comparaţie cu celelalte pagini ale site-ului dvs. Acest atribut nu are niciun efect asupra modului în care compară Google pagina dvs. cu paginile altor site-uri, ci doar ajută Google să ştie ce pagini de pe site-ul dvs. credeţi că sunt cele mai importante.

    Listă de adrese URL

    Localizaţi următoarea secţiune: 

    <!-- ** MODIFY or DELETE **
"urllist" nodes name text files with lists of URLs. 
An example file "example_urllist.txt" is provided. 

Required attributes: 
path - path to the file 

Optional attributes: 
encoding - encoding of the file if not US-ASCII 
--> 
<urllist path="example_urllist.txt" encoding="UTF-8" />

    Utilizaţi acest format pentru a preciza calea şi numele fişierului text care conţine lista dvs. de adrese URL. Puteţi să utilizaţi fişierul exemplu_listaurl.txt furnizat, ca şablon pentru acel fişier text. Puteţi specifica fie o cale relativă, fie una completă, către serverul dvs. de Web. De exemplu, dacă Generatorul de sitemap-uri şi fişierul urlist.txt se află în acelaşi director, puteţi să specificaţi pur şi simplu numele fişierului .txt. Când creaţi un fişier text cu o altă codificare decât UTF-8, puteţi utiliza atributul encoding pentru a indica această codificare. Dacă aveţi mai multe fişiere .txt, puteţi utiliza metacaractere. De exemplu:

    <urllist path="example_urllist*.txt" encoding="UTF-8" />

    Pentru fiecare adresă URL pe care o includeţi în fişierul text, puteţi să specificaţi data ultimei modificări, frecvenţa de modificare şi prioritatea. Consultaţi secţiunea referitoare la fişierul text URLlist pentru informaţii complete despre structura acestui fişier.

    Căi de directoare

    Localizaţi următoarea secţiune: 

    <!-- ** MODIFY or DELETE ** 
"directory" nodes tell the script to walk the file system and 
include all files and directories in the Sitemap.

Required attributes:
path - path to begin walking from 
url - URL equivalent of that path 

Optional attributes:
default_file - name of the index or default file for directory URLs

--> 
 <directory  path="/var/www/icons"    url="http://www.example.com/images/" />
      <directory
         path="/var/www/docroot"
         url="http://www.exemplu.ro/"
        default_file="index.html"
      />

    Această secţiune oferă două exemple. Dacă toate paginile dvs. sunt cuprinse în subdirectoare ale unei singure căi, nu trebuie să includeţi decât o singură intrare. Cu toate acestea, dacă aveţi mai multe căi către paginile de pe site-ul dvs., includeţi câte o intrare pentru fiecare.

    Reţineţi că fiecare adresă URL trebuie să înceapă cu adresa URL de bază pe care aţi specificat-o la pasul 3. De exemplu, ambele exemple date în fişierul exemplu_config.xml au adrese URL care încep cu http://www.exemplu.ro/. De aceea, ambele adrese URL sunt corecte.

    Înlocuiţi intrările din exemple cu intrările pentru site. Multe site-uri vor avea doar o singură intrare care duce la adresa URL de bază. Asiguraţi-vă că valoarea path este calea completă spre directorul de pe serverul dvs. de Web. Asiguraţi-vă că valoarea url este adresa URL completă, inclusiv protocolul (cum ar fi http) şi un caracter slash de final, dacă este necesar.

    Puteţi utiliza parametrul default_file pentru a specifica numele fişierului pe care-l utilizează serverul dvs. ca pagină implicită pentru un director. În exemplul de mai sus, /var/www/docroot devine http://www.exemplu.ro/index.html. Nu vi se cere să specificaţi aceasta. Cu toate acestea, dacă faceţi aceasta, Generatorul de sitemap-uri va include pagina care indică spre fiecare subdirector doar o singură dată (în loc să listeze atât adresa URL a directorului, cât şi adresa URL a numelui fişierului) şi va utiliza ultima dată de modificare a fişierului (în loc de a directorului) pentru a extrage atributul lastmod pentru acea pagină.

    Fişiere jurnal de acces

    Localizaţi următoarea secţiune: 

    <!-- ** MODIFY or DELETE **
"accesslog" nodes tell the script to scan webserver log files to
extract URLs on your site.  Both Common Logfile Format (Apache's default 
logfile) and Extended Logfile Format (IIS's default logfile) can be read.
				
Required attributes:
 path - path to the file
Optional attributes:
 encoding - encoding of the file if not US-ASCII
  -->
<accesslog path="/etc/httpd/logs/access.log" encoding="UTF-8" />
<accesslog path="/etc/httpd/logs/access.log.0" encoding="UTF-8" />
<accesslog path="/etc/httpd/logs/access.log.1.gz" encoding="UTF-8" />

    Această secţiune oferă trei exemple. Ar trebui să înlocuiţi aceste intrări şi să includeţi o intrare pentru fiecare fişier jurnal. Asiguraţi-vă că valoarea path este calea completă şi numele de fişier de pe serverul dvs. de Web. Dacă fişierele jurnal nu sunt codificate în US-ASCII sau UTF-8, atunci utilizaţi atributul opţional de codificare pentru a specifica codificarea. În loc să listaţi fiecare fişier jurnal, puteţi utiliza metacaractere. De pildă, în exemplul de mai sus, aţi putea include următoarea intrare, care ar include toate trei fişierele jurnal:

    <accesslog path="/etc/httpd/logs/access.log*" encoding="UTF-8" />

    Generatorul de sitemap-uri acordă prioritate adreselor URL pe care le găseşte în fişierele jurnal, după cât de des este accesată fiecare adresă URL. De exemplu, unei adrese URL care a fost accesată de 100 de ori i se va da o prioritate mai mare decât unei adrese URL care a fost accesată de două ori. Desemnarea reală de prioritate este relativă şi depinde de fiecare adresă URL, în comparaţie cu celelalte adrese URL din site.

    sitemap

    Localizaţi următoarea secţiune: 

     <!-- ** MODIFY or DELETE **
        
"sitemap" nodes tell the script to scan other Sitemap files.  This can    
be useful to aggregate the results of multiple runs of this script into
a single Sitemap.
				 
Required attributes:
  path - path to the file
  -->
<sitemap path="/var/www/docroot/subpath/sitemap.xml" />

    Această secţiune oferă un exemplu. Ar trebui să înlocuiţi această intrare şi să includeţi o intrare pentru fiecare sitemap pe care doriţi să-l includeţi. Asiguraţi-vă că valoarea path este calea completă şi numele fişierului de pe serverul dvs. de Web. De asemenea, puteţi lista sitemap-urile arhivate cu gzip, dacă au o extensie .gz. În loc să listaţi fiecare sitemap, puteţi utiliza metacaractere. De exemplu, următoarea intrare ar include orice sitemap-uri care încep cu cuvântul „sitemap” şi au o extensie .xml:

    <sitemap path="/var/www/docroot/subpath/sitemap*.xml" />

    Generatorul de sitemap-uri extrage toate adresele URL şi datele opţionale listate pentru fiecare adresă URL pentru fiecare sitemap pe care-l listaţi şi creează un sitemap cu aceste informaţii. Deocamdată, nu putem garanta că această metodă va funcţiona cu sitemap-uri create cu alte instrumente decât Generatorul de sitemap-uri.

  8. Localizaţi secţiunea de definire a filtrului:
    9.  
    <!-- ********************************************************         
FILTERS
				
Filters specify wild-card patterns that the script compares
against all URLs it finds. Filters can be used to exclude
certain URLs from your Sitemap, for instance if you have
hidden content that you hope the search engines don't find.

Filters can be either type="wildcard", which means standard
path wildcards (* and ?) are used to compare against URLs,
 or type="regexp", which means regular expressions are used
to compare.

Filters are applied in the order specified in this file.
An action="drop" filter causes exclusion of matching URLs.
An action="pass" filter causes inclusion of matching URLs,
shortcutting any other later filters that might also match.
If no filter at all matches a URL, the URL will be included.
Together you can build up fairly complex rules.

The default action is "drop".
The default type is "wildcard".

You can MODIFY or DELETE these entries as appropriate for
your site. However, unlike above, the example entries in
this section are not contrived and may be useful to you as
they are.
********************************************************* -->

<!-- Exclude URLs that end with a '~' (IE: emacs backup files) -->
<filter action="drop" type="wildcard" pattern="*~" />

<!-- Exclude URLs within UNIX-style hidden files or directories -->
<filter action="drop" type="regexp" pattern="/\.[^/]*" />

    Puteţi utiliza filtrarea pentru a exclude adrese URL specifice din sitemap-ul generat. E posibil să doriţi să faceţi aceasta pentru a crea o listă mai curată, pentru a reduce listările redundante sau pentru a împiedica anumite adrese URL să fie indexate. Reţineţi că, dacă utilizaţi un fişier robots.txt pentru a împiedica adresele URL să fie indexate, atunci chiar dacă adresele URL sunt incluse în sitemap-ul dvs., Google nu le va căuta sau indexa.

    Puteţi utiliza oricare sau toate metodele de filtrare. Puteţi şterge intrările de care nu aveţi nevoie şi puteţi crea intrări suplimentare, dacă doriţi. Mai jos sunt exemple de utilizări. 

    <filter action="drop" type="wildcard" pattern="*.jpg" />

    Acest filtru exclude adrese URL care se termină în .jpg. Puteţi include un filtru similar dacă toate imaginile site-ului dvs. sunt încastrate în pagini HTML şi nu ar trebui accesate ca adrese URL de sine stătătoare. 

    <filter action="pass" type="wildcard" pattern="*.htm*" /><filter action="drop" type="wildcard" pattern="*" />
    Acest filtru include toate fişierele .htm*, dar exclude orice altceva.

Referinţă de sintaxă pentru fişierul de configuraţie

Mai jos este o explicaţie completă a sintaxei fişierului de configuraţie. Fiecare etichetă începe cu o mostră de cod, urmată de o descriere a atributelor.

site
Etichetă obligatorie la începutul fiecărui fişier de configuraţie. 

<site base_url="http://www.exemplu.ro/"
store_into="/var/www/html/sitemap.xml.gz"
verbose="1"
supress_search_engine_notify="1"
default_encoding="UTF-8">

base_url
 obligatorie Calea HTTP a bazei site-ului dvs. de Web - numai adresele URL care încep cu această bază pot fi incluse în sitemap 
store_into
 obligatorie Calea serverului de Web către fişierul de ieşire dorit. Script-ul va crea acest fişier - nu este necesar să creaţi fişierul înainte de a executa script-ul. 
verbose
 opţională Introduceţi un număr de la 0 la 3, numerele mai mari corespunzând informaţiilor de depanare mai detaliate 
suppress_search_engine_notify
 opţională Dezactivaţi notificările motorului de căutare introducând „1” pentru testare 
default_encoding
 opţională Specificaţi o codificare de caractere care să se aplice căilor fişierelor de sistem şi adreselor URL

url
Etichetă opţională pe care o puteţi utiliza pentru a lista fiecare adresă URL din site-ul dvs. 

<url
href="http://www.exemplu.ro/stats?q=varsta" 
lastmod="2004-11-14T01:00:00-07:00" 
changefreq="yearly" 
priority="0.3"
/> 

href
 obligatorie Calea HTTP a bazei site-ului dvs. de Web - numai adresele URL care încep cu această bază pot fi incluse în sitemap 
lastmod
 opţională Ora la care adresa URL a fost modificată ultima dată, în format Datetime W3C (AAAA-LL-ZZThh:mm:ss+00:00). Puteţi să omiteţi ora. Exemple:
  „2005-02-21T18:00:15+00:00”
  „2005-02-21”		 
changefreq
 opţională Frecvenţa probabilă de modificare a adresei URL. Aceasta este considerată o sugestie, nu o comandă. Valoarea trebuie să fie una dintre următoarele: „always” (întotdeauna), „hourly” (în fiecare oră), „daily” (zilnic), „weekly” (săptămânal), „monthly” (lunar), „yearly” (anual) sau „never” (niciodată). 
priority
 opţională Prioritatea acestei pagini faţă de alte pagini de pe acelaşi site. Valoarea este un număr între 0.0 şi 1.0, unde 0.0 este cea mai joasă prioritate şi 1.0 este cea mai înaltă prioritate. Prioritatea poate afecta ordinea în care selectează motoarele de căutare adresele URL de pe site-ul dvs. pentru a le explora. De vreme ce prioritatea este relativă, este utilizată doar pentru a selecta dintre adresele URL din site-ul dvs.; prioritatea paginilor dvs. nu va fi comparată cu prioritatea paginilor de pe alte site-uri.

urllist
Etichetă opţională pe care o puteţi utiliza pentru a indica un fişier text care conţine o listă a adreselor URL din site-ul dvs. 

<urllist path="/var/www/html/urllist.txt" encoding="UTF-8" />

path
 obligatorie Calea şi numele de fişier pentru fişierul .txt. Puteţi specifica fie o cale relativă, fie una completă. 
encoding
 opţională Codificarea fişierului, dacă nu este UTF-8.

Fişierul urllist.txt este un fişier text simplu care conţine o listă de adrese URL de mapat. De asemenea, puteţi include atribute opţionale pentru fiecare adresă URL. Atributele sunt introduse pe acelaşi rând cu adresa URL şi sunt separate printr-un singur spaţiu. De exemplu: 

http://www.exemplu.ro/abc/ceva
http://www.exemplu.ro/abc/xyy.pdf lastmod=2001-12-31T14:05:06+00:00
http://www.exemplu.ro/abc/def?x=12&y=23 changefreq=weekly priority=0.3

lastmod
 opţională

Ora la care a fost modificată ultima dată adresa URL, în format Datetime W3C (AAAA-LL-ZZThh:mm:ss+00:00). Puteţi să omiteţi ora. Exemple:

  „2005-02-21T18:00:15+00:00”
  „2005-02-21”

 
changefreq
 opţională Frecvenţa probabilă de modificare a adresei URL. Aceasta este considerată o sugestie, nu o comandă. Valoarea trebuie să fie una dintre următoarele: „always” (întotdeauna), „hourly” (în fiecare oră), „daily” (zilnic), „weekly” (săptămânal), „monthly” (lunar), „yearly” (anual) sau „never” (niciodată). 
priority
 opţională Prioritatea acestei pagini faţă de alte pagini de pe acelaşi site. Valoarea este un număr între 0.0 şi 1.0, unde 0.0 este cea mai joasă prioritate şi 1.0 este cea mai înaltă prioritate. Prioritatea poate afecta ordinea în care selectează motoarele de căutare adresele URL de pe site-ul dvs. pentru a le explora. Deoarece prioritatea este relativă, se utilizează numai pentru selectarea dintre adresele URL de pe site-ul dvs.; prioritatea paginilor dvs. nu va fi comparată cu prioritatea paginilor de pe alte site-uri.

directory
Etichetă opţională pe care o puteţi utiliza pentru a specifica directoarele din site-ul dvs. pentru ca Generatorul de sitemap-uri să poată crea o listă de adrese URL din fişierele găsite în acele directoare. 

-->  <directory  path="/var/www/icons"    url="http://www.exemplu.ro/imagini/" />
  <directory
     path="/var/www/docroot"
     url="http://www.exemplu.ro/"
    default_file="index.html"
  />
 
path
 obligatorie Precizează calea iniţială. Generatorul de sitemap-uri va traversa acest director şi toate subdirectoarele. 
url
 obligatorie Specifică adresa URL echivalentă valorii pentru cale. 
default_file
 opţională Specifică fişierul implicit pentru un director de pe server.

accesslog
Etichetă opţională pe care o puteţi utiliza pentru a specifica o cale şi un nume de fişier pentru jurnalele de acces de tip IIS şi Apache, astfel încât Generatorul de sitemap-uri să poată prelua automat adrese URL din ele. 

<accesslog path="/etc/httpd/logs/access-0.log" encoding="UTF-8"/>
path
 obligatorie Precizează calea către fişier. 
encoding
 opţională Precizează codificarea fişierului, dacă nu este UTF-8.

sitemap
Etichetă opţională pe care o puteţi utiliza pentru a preciza calea şi numele fişierului pentru sitemap-urile existente pe care le-aţi creat cu Generatorul de sitemap-uri. Generatorul de sitemap-uri va crea un singur sitemap care include adresele URL conţinute în fiecare sitemap. 

<sitemap path="/var/www/docroot/subpath/sitemap.xml" />
path
 obligatorie Menţionează calea către fişierul sitemap.

filter
Etichetă opţională pe care o puteţi utiliza pentru a construi reguli care includ sau exclud fişiere specifice. Filtrele se respectă în ordinea în care apar în fişierul config.xml. Cu toate acestea, amestecarea intrărilor de filtru şi intrărilor de admisie (url, urllist, directory, sau accesslog) nu are niciun efect suplimentar - fiecare adresă URL pe care o adaugă Generatorul de sitemap-uri la sitemap este întâi comparată cu fiecare filtru. Dacă niciun filtru nu se potriveşte cu o adresă URL, se include implicit adresa URL în sitemap.

<filter action="drop" type="wildcard" pattern="*/internal/*" />
action
 opţională

Acţiunea pe care trebuie să o realizeze filtrul. Valorile corecte sunt:
  • drop - exclude adresele URL care se potrivesc. Aceasta este acţiunea implicită, deci, dacă nu aţi specificat nicio acţiune, generatorul presupune că este „drop”.
  • pass - include adresele URL care se potrivesc.

 
type
 opţională

Tipul de filtrare. Valorile corecte sunt:
  • wildcard - metacaracterele standard pentru cale (? şi *) se utilizează pentru compararea cu adresele URL. Acesta este tipul implicit, deci, dacă nu se specifică niciun tip, generatorul presupune că este „wildcard”.
  • regexp - se utilizează expresii obişnuite.
 
pattern
 obligatorie Specifică modelul cu care să se potrivească.

Codificări

Fişierele la care face referire fişierul de configuraţie, fie listele de adrese URL, fie fişierele jurnal ale serverului de Web, pot utiliza alte codificări decât cea implicită UTF-8. Puteţi specifica codificări alternative în config.xml pentru a influenţa modul în care Generatorul de sitemap-uri citeşte fişierele dvs. Unele dintre codificările obişnuite sunt:

  • encoding=„utf-8” este presupusă ca implicită
  • encoding=„ascii” este un subset de UTF-8, deci nu este obligatoriu să îl specificaţi
  • encoding=„iso-8859-1” este uzuală pentru multe limbi vest-europene
2b. Informaţii suplimentare pentru crearea unui fişier de configuraţie pentru mobil

Creaţi un fişier de configuraţie pentru un sitemap mobil în acelaşi mod în care creaţi un fişier pentru un sitemap care nu este mobil. Cu toate acestea, trebuie să creaţi un fişier de configurare separat pentru fiecare limbaj de markup şi să executaţi Generatorul de sitemap-uri cu fiecare fişier de configuraţie separat, pentru a putea crea un sitemap separat pentru fiecare.

Fiecare fişier de configuraţie trebuie să:

  • Specifice un nume de fişier diferit pentru valoarea store_into .
  • Utilizeze filtre pentru a specifica adresele URL care să fie excluse sau incluse în limbajul de markup. Reţineţi că fiecare sitemap ar trebui să includă adrese URL pentru un singur limbaj de markup. Aceasta înseamnă că aceeaşi adresă URL poate fi inclusă în mai multe sitemap-uri, dacă acele adrese URL utilizează mai multe limbaje de markup.

Exemple de filtrare

Mai jos sunt câteva exemple despre cum puteţi utiliza filtre bazate pe extensii pentru a genera sitemap-uri mobile pentru diferite limbaje de markup. Filtrarea specifică pe care o utilizaţi ar trebui să fie bazată pe tipurile de limbaje de markup utilizate în site-ul dvs. şi pe modul în care specificaţi fiecare tip. Dacă aţi implementat diferit detaliile site-ului dvs. (de exemplu, puteţi organiza adresele URL cu diferite limbaje de markup în foldere separate), ar trebui să filtraţi în funcţie de caracteristicile implementărilor site-ului dvs. Reţineţi că filtrele se aplică în ordinea în care le listaţi în fişierul de configuraţie. Astfel, primul filtru pe care trebuie să îl listaţi este o acţiune „pass” care specifică adresele URL pe care doriţi să le includeţi în sitemap.

Pentru a crea un sitemap pentru conţinut WML (WAP 1.2):

<filter action="pass" type="wildcard" pattern="*.wml" />
<filter action="drop" type="wildcard" pattern="*.*" />

Pentru a crea un sitemap pentru conţinutul de profil mobil XHTML (WAP 2.0):

<filter action="pass" type="wildcard" pattern="*.xhtml" />
<filter action="drop" type="wildcard" pattern="*.*" />
3. Încărcarea fişierelor pe serverul dvs. de Web

Ar trebui să încărcaţi următoarele fişiere pe serverul dvs. de Web într-o locaţie pe care o puteţi accesa dintr-o linie de comandă:

  • config.xml —acesta este fişierul de configuraţie pe care tocmai l-aţi creat cu example_config.xml.
  • sitemap_gen.py —acesta este script-ul Python care generează sitemap-ul dvs.
  • urllist.txt —acest fişier este opţional; trebuie să-l includeţi doar dacă aţi utilizat metoda fişier text de generare a unui sitemap.

Metoda pe care o utilizaţi pentru a încărca aceste fişiere depinde de mediul dvs. Metodele uzuale includ FTP şi SCP. Pentru informaţii suplimentare, contactaţi gazda dvs. Web.

5. Trimiterea sitemap-ului dvs. către Google

Generatorul de sitemap-uri creează un fişier sitemap.xml.gz în locaţia pe care aţi specificat-o în fişierul de configuraţie. Odată ce acest fişier a fost creat cu succes, asiguraţi-vă că este accesibil printr-un browser de Web. Apoi, adăugaţi-l la contul dvs. Sitemap-uri Google. Aceasta permite Google să vă furnizeze informaţii utile referitoare la stare şi statistici. Dacă Google raportează probleme referitoare la sitemap-ul dvs., puteţi corecta problema şi îl puteţi retrimite. Trebuie să adăugaţi sitemap-ul manual doar o singură dată. După aceea, puteţi utiliza o solicitare HTTP pentru a informa Google despre modificările sitemap-ului dvs. (deşi îl puteţi retrimite şi prin contul dvs. de instrumente Google pentru webmasteri).

6. Setarea unui script recurent

Vă sugerăm să configuraţi Generatorul de sitemap-uri pentru a se executa la fel de des cu toate acestea se modifică conţinutul, până la o frecvenţă maximă de o dată pe oră.

Webmasterii cu un server de Web UNIX pot lua în considerare setarea ca o planificare cron.

Webmasterii care utilizează alte platforme trebuie să-şi contacteze administratorul de sistem pentru asistenţă în configurarea de script-uri recurente. Puteţi, de asemenea, beneficia de sfaturi amănunţite în Grupul Sitemap-uri Google la Grupuri Google.

Puteţi utiliza o solicitare HTTP pentru a anunţa Google despre modificările sitemap-ului dvs. Cu toate acestea, asiguraţi-vă că vă conectaţi la instrumentele Google pentru webmasteri cu contul dvs. Google pentru a adăuga sitemap-ul manual la contul dvs. Google de instrumente pentru webmasteri.

Depanare

În această secţiune vorbim despre unele întrebări sau probleme comune pe care le întâmpină oamenii când utilizează Generatorul de sitemap-uri şi ce puteţi face dacă întâlniţi una dintre acestea.

Accesibil pe Web

Problema: Generatorul de sitemap-uri returnează următoarea eroare şi avertizare:

 
[ERROR] When attempting to access your generated Sitemap at the following URL: http://www.exemplu.ro/sitemap.xml.gz
	we failed to read it.  Please verify the store_into path you specified in your configuration file is web-accessible.  Consult the FAQ for more information.
	[WARNING] Proceeding to notify with an unverifiable URL.

Ce s-a întâmplat? Generatorul de sitemap-uri a creat fişierul unde aţi specificat, apoi a încercat să-l găsească utilizând HTTP aşa cum ar face-o motorul de căutare şi a eşuat. A mers înainte şi a notificat oricum motoarele de căutare, dar scoate în evidenţă eroarea şi avertizarea pentru a vă înştiinţa că sitemap-ul dvs. e posibil să nu poată fi citit de motoarele de căutare.

Generatorul de sitemap-uri creează fişierul sitemap pe calea specificată în atributul store_into al fişierului config.xml. Apoi, Generatorul de sitemap-uri generează o adresă URL spre acel fişier utilizând atributul base_url şi raportează adresa URL motoarelor de căutare. De exemplu, dacă setaţi configuraţia pe:

 
<site base_url="http://www.exemplu.ro/"
	      store_into="/var/www/html/sitemap.xml.gz">

Generatorul de sitemap-uri va anunţa motoarele de căutare să vă caute sitemap-ul la:

	http://www.exemplu.ro/sitemap.xml.gz

Fişierul trebuie să fie accesibil prin această adresă URL. Dacă fişierul nu poate fi găsit utilizând această adresă URL, motoarele de căutare nu au cum să găsească sitemap-ul dvs.

Ce puteţi face? Trebuie să verificaţi dacă fişierul config.xml precizează căile corecte base_url şi store_into pentru sitemap. De asemenea, trebuie să confirmaţi faptul că browserele de Web pot găsi fişierul sitemap din base_url. Dacă găsiţi erori în cale sau în adresa URL, va trebui să executaţi din nou Generatorul de sitemap-uri sau doar să mutaţi fişierul sitemap în locul corect şi să informaţi manual motoarele de căutare prin intermediul site-urilor lor de Web.

Suportul pentru XML

Problema: Generatorul de sitemap-uri dă următoarea eroare:

 
Some installs of Python 2.2 do not include complete support for XML. Please try upgrading your version of Python and re-running the script.

Ce s-a întâmplat? După cum spune mesajul, unele platforme au o versiune de Python (limbajul în care este scris Generatorul de sitemap-uri) căreia îi lipsesc bibliotecile de suport necesare pentru procesarea fişierelor XML. Script-ul necesită suport XML complet pentru a rula.

Ce puteţi face? Încercaţi să actualizaţi versiunea dvs. de Python instalată la o versiune mai nouă. Pentru a face aceasta, va trebui probabil să contactaţi administratorul de sistem.

Reţineţi că acest lucru afectează numai Generatorul de sitemap-uri, nu sitemap-urile în general. Dacă aveţi o altă metodă sau un alt instrument pentru a crea sitemap-uri, bineînţeles că o puteţi utiliza şi trimite sitemap-urile la motoarele de căutare.

Caracterele non-ASCII

Problema: Numele domeniului site-ului dvs. sau adresele URL din el conţin caractere non-ASCII.

În general, adresele URL non-ASCII ar trebui să fie codificate utilizând UTF-8 înainte de a fi codificate cu % ca şi caracter de escape. Cu toate acestea, unele servere de Web răspund corect doar dacă adresele URL sunt codificate utilizând o metodă de codificare diferită de UTF-8. Toate adresele URL din sitemap-urile dvs., inclusiv adresa URL a sitemap-ului dvs., trebuie să fie codificate pentru a putea fi citite de serverul de Web pe care sunt localizate. În secţiunea de definire a site-ului, utilizaţi atributul opţional default_encoding pentru a specifica codificarea utilizată de serverul dvs. de Web. Dacă nu utilizaţi această etichetă, iar serverul dvs. de Web utilizează o codificare diferită de UTF-8, nu putem şti ce codificare să utilizăm.

Dacă adresa dvs. URL conţine caractere non-ASCII, vă recomandăm să executaţi script-ul Generator de sitemap-uri utilizând Python 2.3 sau o versiune mai nouă. Această versiune de Python are suport mărit pentru non-ASCII. Dacă numele domeniului dvs. conţine caractere non-ASCII, trebuie să utilizaţi Python 2.3 sau o versiune mai nouă deoarece suportul pentru Internationalizing Domain Names in Applications (Internaţionalizarea numelor de domenii în aplicaţii) (IDNA) nu a fost adăugat până la această versiune. Fără suportul IDNA, Generatorul de sitemap-uri nu poate codifica corect un nume de domeniu non-ASCII.
