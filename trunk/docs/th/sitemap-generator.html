ก่อนที่คุณจะเริ่มต้น

เครื่องมือสร้าง Sitemap ของ Google เป็นสคริปต์ Python ที่สร้าง Sitemap สำหรับเว็บไซต์ของคุณโดยใช้โพรโทคอล Sitemap สคริปต์นี้สามารถสร้าง Sitemap จากรายการ URL ไดเรกทอรีของเว็บเซิร์ฟเวอร์ หรือจากไฟล์บันทึกการเข้าถึง ในการใช้งานสคริปต์นี้:

  • คุณต้องสามารถเชื่อมต่อและเรียกใช้สคริปต์บนเว็บเซิร์ฟเวอร์ของคุณได้
  • เว็บเซิร์ฟเวอร์ของคุณต้องติดตั้ง Python 2.2 ขึ้นไป
  • คุณต้องทราบคำสั่งที่เปิดใช้ Python (โดยทั่วไปก็คือ Python แต่อาจแตกต่างกันตามการติดตั้ง ตัวอย่างเช่น หากเว็บเซิร์ฟเวอร์ติดตั้ง Python สองรุ่น รุ่นเก่าอาจเรียกได้จากคำสั่ง Python และรุ่นใหม่อาจเรียกได้จากคำสั่ง Python2)
  • คุณต้องทราบเส้นทางไดเรกทอรีไปยังเว็บไซต์ของคุณ หากเว็บเซิร์ฟเวอร์ของคุณโฮสต์เพียงหนึ่งเว็บไซต์ เส้นทางอาจเป็น var/www/html หากคุณมีเซิร์ฟเวอร์เสมือนที่โฮสต์หลายเว็บไซต์ เส้นทางอาจเป็นเส้นทาง home/virtual/site1/fst/var/www/html
  • คุณต้องสามารถอัปโหลดไฟล์ไปยังเว็บเซิร์ฟเวอร์ของคุณได้ (ตัวอย่างเช่น การใช้ FTP)
  • หากคุณจะสร้างรายการ URL โดยใช้ไฟล์บันทึกการเข้าถึง คุณต้องทราบการเข้ารหัสที่ใช้สำหรับไฟล์บันทึกเหล่านั้นและเส้นทางแบบสมบูรณ์ไปยังไฟล์บันทึกเหล่านั้น

หากคุณไม่แน่ใจเกี่ยวกับสิ่งเหล่านี้ คุณสามารถตรวจสอบกับบริษัทผู้ให้บริการพื้นที่เว็บ

ตอนนี้ คุณพร้อมที่จะเริ่มต้นแล้ว ต่อไปนี้คือภาพรวมของสิ่งที่คุณจะต้องทำ

  1. ดาวน์โหลด โปรแกรมไฟล์ของเครื่องมือสร้าง Sitemap แยกไฟล์ลงในไดเรกทอรีในเครื่อง
  2. สร้างไฟล์กำหนดค่า สำหรับเว็บไซต์โดยใช้ไฟล์ example_config.xml ที่ให้เป็นเทมเพลต แก้ไขไฟล์นี้ตามต้องการสำหรับเว็บไซต์ของคุณและบันทึกไฟล์
  3. อัปโหลดไฟล์ที่จำเป็น ไปยังเว็บเซิร์ฟเวอร์
  4. เรียกใช้ sitemap_gen.py
  5. เพิ่ม Sitemap ที่สร้างขึ้นลงในบัญชี Google Webmaster Tools
  6. ตั้งค่าสคริปต์ที่เกิดขึ้นประจำ (ตัวเลือก)

หากคุณไม่สามารถใช้เครื่องมือสร้าง Sitemap คุณสามารถเพิ่ม Sitemap ลงในบัญชี Google Webmaster Tools ใน รูปแบบอื่น เช่น ไฟล์ข้อความแบบง่าย โปรแกรมอื่นที่สนับสนุนโพรโทคอล Sitemap

สำหรับ News Sitemaps: ไม่แนะนำให้ใช้เครื่องมือสร้าง Sitemap ในการสร้าง Google News Sitemaps ในขณะนี้ เนื่องจากข้อกำหนดพิเศษของ News Sitemaps News Sitemaps ออกแบบไว้สำหรับรายการที่มีการเปลี่ยนแปลงตลอดเวลา เช่น บทความข่าวที่เพิ่งเผยแพร่ล่าสุดเท่านั้น (ไม่ใช่ทั้งเว็บไซต์) ซึ่งจะมีการอัปเดตบ่อยๆ

2a. การสร้างไฟล์กำหนดค่า

ในส่วนนี้จะมีคำแนะนำทีละขั้นตอนในการสร้างไฟล์กำหนดค่า นอกจากนี้ ยังมี ข้อมูลอ้างอิงทั้งหมด สำหรับตัวเลือกที่มีอยู่ หากคุณสร้าง Mobile Sitemap โปรดดู คำแนะนำเกี่ยวกับมือถือเพิ่มเติม

ในการสร้างไฟล์กำหนดค่าสำหรับเว็บไซต์ของคุณ คุณต้องมีข้อมูลต่อไปนี้:

  • URL พื้นฐานสำหรับเว็บไซต์ (เช่น http://www.example.co.th/) ให้แน่ใจว่าคุณระบุโพรโทคอล (เช่น http://) ตัวอย่างเช่น http://www.google.co.th เป็น url พื้นฐานที่ถูกต้อง แต่ www.google.co.th ไม่ถูกต้อง
  • เส้นทางของเว็บเซิร์ฟเวอร์ไปยังตำแหน่งที่คุณต้องการจัดเก็บ Sitemap โดยทั่วไป นี่คือเส้นทางไปยัง URL พื้นฐาน เนื่องจาก Sitemap ไม่สามารถมี URL ที่อยู่ในไดเรกทอรีระดับบนสุดจากตำแหน่งของ Sitemap ได้ เมื่อคุณเรียกใช้เครื่องมือสร้าง Sitemap ของ Google ก็จะสร้าง Sitemap และวาง Sitemap ในตำแหน่งที่คุณระบุ
  • วิธีที่คุณต้องการให้เครื่องมือสร้าง Sitemap ใช้เพื่อสร้าง Sitemap ของคุณ คุณสามารถใช้วิธีการใดๆ ผสมกัน ต่อไปนี้คือวิธีการที่สามารถใช้ได้:
    • URL —จะแสดง URL แต่ละรายการในส่วนของไฟล์กำหนดค่าควบคู่กับข้อมูลเกี่ยวกับ URL แต่ละรายการ โดยทั่วไป คุณสามารถใช้วิธีนี้ร่วมกับวิธีอื่นเพื่อเพิ่ม URL ซึ่งวิธีอื่นไม่สามารถทำได้
    • รายการ URL —จะชี้ไฟล์กำหนดค่าไปยังไฟล์ข้อความที่มีรายการ URL คุณอาจต้องใช้วิธีนี้หากมีไฟล์ข้อความนี้อยู่แล้ว หรือหากคุณใช้สคริปต์ในการสร้างรายการ URL
    • เส้นทางไดเรกทอรี—จะระบุเส้นทางไดเรกทอรีสำหรับเว็บไซต์ของคุณและ URL ที่ไปยังเส้นทางเหล่านั้น เครื่องมือสร้าง Sitemap จะสร้างรายการ URL ตามเนื้อหาของไดเรกทอรีเหล่านั้น คุณอาจต้องใช้วิธีนี้หากเว็บไซต์ของคุณประกอบด้วยไฟล์ HTML แบบคงที่
    • ไฟล์บันทึกการเข้าถึง—จะชี้ไปที่เส้นทางไปยังไฟล์บันทึกของคุณ เครื่องมือสร้าง Sitemap จะสร้างรายการ URL ตาม URL ที่อยู่ในไฟล์บันทึก คุณอาจต้องใช้วิธีนี้หากเว็บไซต์ของคุณประกอบด้วยหน้าเว็บแบบไดนามิก
    • Sitemap—จะชี้ไปยัง Sitemap ที่มีอยู่ ซึ่งคุณสร้างด้วยเครื่องมือสร้าง Sitemap เครื่องมือสร้าง Sitemap จะสร้าง Sitemap รายการเดียวที่รวม URL ที่มีใน Sitemap แต่ละรายการ คุณสามารถใช้วิธีนี้หากคุณได้สร้าง Sitemap ขนาดเล็กไว้หลายรายการแล้ว ซึ่งคุณต้องการรวมเป็น Sitemap ขนาดใหญ่เพียงรายการเดียว

สร้างไฟล์กำหนดค่าดังนี้:
  1. เปิดไฟล์ example_config.xml ในโปรแกรมแก้ไขข้อความ บันทึกเป็นไฟล์ใหม่ (เช่น config.xml หรือ mysite_config.xml)
    2.
  2. ค้นหาส่วนคำนิยามของเว็บไซต์: 
    3. <site base_url="http://www.example.co.th/" store_into="/var/www/docroot/sitemap.xml.gz"
verbose="1">
  3. เปลี่ยนค่า base_url เป็น URL สำหรับเว็บไซต์ของคุณ
  4. เปลี่ยนค่า store_into เป็นเส้นทางบนเว็บเซิร์ฟเวอร์ที่คุณต้องการจัดเก็บ Sitemap และชื่อไฟล์ที่คุณต้องการใช้สำหรับ Sitemap โดยทั่วไป นี่คือเส้นทางไปยัง URL พื้นฐาน เนื่องจาก Google สามารถยอมรับเฉพาะ URL ที่อยู่ในระดับเดียวกันหรือไดเรกทอรีย่อยของไดเรกทอรีที่เก็บ Sitemap คุณสามารถระบุเส้นทางแบบสัมพัทธ์จากไดเรกทอรีที่คุณอัปโหลดสคริปต์หรือเส้นทางแบบสมบูรณ์จากรากของเว็บเซิร์ฟเวอร์ หากคุณอัปโหลดสคริปต์ไปยังไดเรกทอรี URL พื้นฐานของคุณ คุณก็สามารถระบุชื่อไฟล์ได้
    5.
  5. ค้นหาส่วนวิธีสร้างที่ขึ้นต้นด้วย ** MODIFY หรือ DELETE ** ส่วนต่างๆ เหล่านี้เป็นส่วนหนึ่งในวิธีสร้าง Sitemap
  6. ลบส่วนสำหรับวิธีที่คุณไม่ต้องการใช้อีกต่อไป
  7. ทำตามคำแนะนำด้านล่างสำหรับวิธีที่คุณต้องการใช้

    8. URL

    ค้นหาส่วนต่อไปนี้: 

    <!-- ** MODIFY or DELETE ** 
"url" nodes specify individual URLs to include in the map. <br>

Required attributes: 
href - the URL

Optional attributes: 
lastmod - timestamp of last modification (ISO8601 format) 
changefreq - how often content at this URL is usually updated
priority - value 0.0 to 1.0 of relative importance in your site 
--> 

<url href="http://www.example.co.th/stats?q=name" /> 
<url 
href="http://www.example.co.th/stats?q=age" 
lastmod="2004-11-14T01:00:00-07:00" 
changefreq="yearly" 
priority="0.3"
/>

    ส่วนนี้มีสองตัวอย่าง: ตัวอย่างแรกจะประกอบด้วยแอตทริบิวต์ที่จำเป็นเท่านั้น และส่วนที่สองจะประกอบด้วยแอตทริบิวต์ที่จำเป็นและแอตทริบิวต์ตัวเลือก

    ใช้รูปแบบนี้สำหรับ URL แต่ละรายการที่คุณต้องการรวม แอตทริบิวต์ changefreq จะให้ข้อมูลแก่ Google เกี่ยวกับความถี่ในการอัปเดต URL ซึ่งข้อมูลนี้จะช่วยให้ Google ทราบความถี่ในการเข้าชมหน้าเว็บสำหรับเนื้อหาใหม่ แอตทริบิวต์ priority จะให้ข้อมูลแก่ Google เกี่ยวกับความสำคัญของหน้าเว็บนี้เปรียบเทียบกับหน้าเว็บอื่นๆ ในเว็บไซต์ของคุณ แอตทริบิวต์นี้ไม่มีผลกับวิธีการที่ Google ใช้เปรียบเทียบหน้าเว็บของคุณกับหน้าเว็บในเว็บไซต์อื่นๆ แต่จะช่วยให้ Google ทราบว่าหน้าเว็บใดในเว็บไซต์ที่คุณคิดว่าสำคัญที่สุด

    รายการ URL

    ค้นหาส่วนต่อไปนี้: 

    <!-- ** MODIFY or DELETE **
"urllist" nodes name text files with lists of URLs. 
An example file "example_urllist.txt" is provided. 

Required attributes: 
path - path to the file 

Optional attributes: 
encoding - encoding of the file if not US-ASCII 
--> 
<urllist path="example_urllist.txt" encoding="UTF-8" />

    ใช้รูปแบบนี้เพื่อชี้ไปยังเส้นทางและชื่อของไฟล์ข้อความที่มีในรายการ URL คุณสามารถใช้ไฟล์ example_urllist.txt ที่ให้มาเป็นเทมเพลตสำหรับไฟล์ข้อความ คุณสามารถระบุเป็นเส้นทางแบบสัมพัทธ์หรือเส้นทางแบบสมบูรณ์เพื่อไปยังเว็บเซิร์ฟเวอร์ของคุณ ตัวอย่างเช่น หากเครื่องมือสร้าง Sitemap และไฟล์ urlist.txt อยู่ในไดเรกทอรีเดียวกัน คุณสามารถระบุชื่อไฟล์ .txt หากคุณสร้างไฟล์ข้อความที่มีการเข้ารหัสอื่นที่ไม่ใช่แบบ UTF-8 คุณสามารถใช้แอตทริบิวต์ encoding เพื่อระบุการเข้ารหัสนี้ หากคุณมีไฟล์ .txt หลายไฟล์ คุณสามารถใช้สัญลักษณ์แทนได้ ตัวอย่าง:

    <urllist path="example_urllist*.txt" encoding="UTF-8" />

    สำหรับ URL แต่ละรายการที่คุณรวมไว้ในไฟล์ข้อความ คุณสามารถระบุวันที่แก้ไขล่าสุด ความถี่ในการเปลี่ยนแปลง และลำดับความสำคัญ โปรดดู ส่วนอ้างอิงไฟล์ข้อความ URLlist สำหรับข้อมูลทั้งหมดเกี่ยวกับโครงสร้างของไฟล์นี้

        default_file="index.html"
      />

    ส่วนนี้มีสองตัวอย่าง หากหน้าเว็บทั้งหมดของคุณอยู่ในไดเรกทอรีย่อยของเส้นทางหนึ่ง คุณจะต้องรวมเพียงหนึ่งรายการเท่านั้น อย่างไรก็ตาม หากคุณมีหลายเส้นทางไปยังหน้าเว็บในเว็บไซต์ของคุณ ให้รวมรายการสำหรับแต่ละเส้นทาง

    โปรดทราบว่า URL แต่ละรายการต้องขึ้นต้นด้วย URL พื้นฐานที่คุณระบุในขั้นตอนที่ 3 ตัวอย่างเช่น ตัวอย่างทั้งสองแบบที่ให้ไว้ในไฟล์ example_config.xml นั้นมี URL ที่ขึ้นต้นด้วย http://www.example.co.th/ ดังนั้น URL ทั้งสองรายการถูกต้อง

    แทนที่รายการตัวอย่างด้วยรายการสำหรับเว็บไซต์ของคุณ เว็บไซต์จำนวนมากจะมีเพียงหนึ่งรายการเท่านั้นที่ชี้ไปยัง URL พื้นฐาน ให้แน่ใจว่าค่า เส้นทาง เป็นเส้นทางแบบสมบูรณ์ไปยังไดเรกทอรีบนเว็บเซิร์ฟเวอร์ของคุณ ให้แน่ใจว่าค่า url เป็น URL ที่ครบถ้วน รวมถึงโพรโทคอล (เช่น http) และเครื่องหมายสแลชในกรณีจำเป็น

    คุณสามารถใช้พารามิเตอร์ default_file เพื่อระบุชื่อไฟล์ที่เซิร์ฟเวอร์ของคุณใช้เป็นหน้าเว็บเริ่มต้นสำหรับไดเรกทอรี ในตัวอย่างข้างต้น /var/www/docroot จะต้องแก้ไขเป็น http://www.example.co.th/index.html คุณไม่จำเป็นต้องระบุข้อมูลนี้ อย่างไรก็ตาม หากคุณระบุเครื่องมือสร้าง Sitemap จะรวมหน้าเว็บที่แมปกับแต่ละไดเรกทอรีย่อยเพียงครั้งเดียวเท่านั้น (ไม่แสดงทั้ง URL ของไดเรกทอรีและ URL ของชื่อไฟล์) และจะใช้วันที่แก้ไขล่าสุดของไฟล์ (ไม่ใช่ไดเรกทอรี) เพื่อดึงแอตทริบิวต์ lastmod สำหรับหน้าเว็บนั้น

    ส่วนนี้มีสามตัวอย่าง คุณควรแทนที่รายการเหล่านี้และรวมรายการสำหรับไฟล์บันทึกแต่ละไฟล์ ให้แน่ใจว่าค่าเส้นทางเป็นเส้นทางแบบสมบูรณ์และชื่อไฟล์บนเว็บเซิร์ฟเวอร์ของคุณ หากไฟล์บันทึกไม่ได้รับการเข้ารหัสแบบ US-ASCII หรือ UTF-8 ให้ใช้แอตทริบิวต์ตัวเลือก encoding เพื่อ ระบุ การเข้ารหัส แทนที่จะแสดงไฟล์บันทึกแต่ละไฟล์ คุณสามารถใช้สัญลักษณ์แทนได้ ในตัวอย่างข้างต้น คุณสามารถรวมรายการต่อไปนี้ซึ่งจะรวมไฟล์บันทึกทั้งสามไฟล์:

    <accesslog path="/etc/httpd/logs/access.log*" encoding="UTF-8" />

    เครื่องมือสร้าง Sitemap จะกำหนดลำดับความสำคัญให้กับ URL ที่พบในไฟล์บันทึกตามความถี่ในการเข้าถึงของ URL แต่ละรายการ ตัวอย่างเช่น URL ที่มีการเข้าถึง 100 ครั้งจะมีลำดับความสำคัญมากกว่า URL ที่มีการเข้าถึงสองครั้ง การกำหนดลำดับความสำคัญที่แท้จริงจะเกี่ยวเนื่องกันและขึ้นอยู่กับ URL แต่ละรายการเมื่อเปรียบเทียบกับ URL อื่นๆ ในเว็บไซต์

    sitemap

    ค้นหาส่วนต่อไปนี้: 

     <!-- ** MODIFY or DELETE **
        
"sitemap" nodes tell the script to scan other Sitemap files.  This can    
be useful to aggregate the results of multiple runs of this script into
a single Sitemap.
				 
Required attributes:
  path - path to the file
  -->
<sitemap path="/var/www/docroot/subpath/sitemap.xml" />

    ส่วนนี้มีหนึ่งตัวอย่าง คุณควรแทนที่รายการนี้และรวมรายการสำหรับ Sitemap แต่ละรายการที่คุณต้องการรวม ให้แน่ใจว่าค่าเส้นทางเป็นเส้นทางแบบสมบูรณ์และชื่อไฟล์บนเว็บเซิร์ฟเวอร์ของคุณ คุณสามารถแสดง Sitemap แบบ gzip ได้เช่นกันหากมีนามสกุลเป็น .gz แทนที่จะแสดง Sitemap แต่ละรายการ คุณสามารถใช้สัญลักษณ์แทนได้ ตัวอย่างเช่น รายการต่อไปนี้จะรวม Sitemap ใดๆ ที่ขึ้นต้นด้วยคำว่า "sitemap" และมีนามสกุลเป็น .xml

    <sitemap path="/var/www/docroot/subpath/sitemap*.xml" />

    เครื่องมือสร้าง Sitemap จะแยก URL ทั้งหมดและข้อมูลตัวเลือกที่แสดงสำหรับ URL แต่ละรายการสำหรับทุกๆ Sitemap ที่คุณแสดงและสร้าง Sitemap หนึ่งรายการสำหรับข้อมูลนี้ ในขณะนี้ เราไม่สามารถรับประกันได้ว่าวิธีนี้จะใช้ได้ผลกับ Sitemap ที่สร้างขึ้นด้วยเครื่องมืออื่นๆ ที่ไม่ใช่เครื่องมือสร้าง Sitemap ของเรา

  8. ค้นหาส่วนคำนิยามของตัวกรอง:
    9.  
    <!-- ********************************************************         
ตัวกรอง
				
ตัวกรองจะระบุรูปแบบสัญลักษณ์แทนซึ่งสคริปต์ใช้เปรียบเทียบกับ URL ทั้งหมดที่พบ ตัวกรองสามารถใช้เพื่อแยก URL ที่ต้องการออกจาก Sitemap ของคุณ ตัวอย่างเช่น หากคุณซ่อนเนื้อหาที่คุณหวังว่าเครื่องมือค้นหาจะไม่พบ

ตัวกรองสามารถเป็นได้ทั้ง type="wildcard" ซึ่งหมายถึงสัญลักษณ์แทนเส้นทางมาตรฐาน (* และ ?) จะใช้เพื่อเปรียบเทียบกับ URL หรือ type="regexp" ซึ่งหมายถึงการแสดงผลปกติจะใช้เพื่อเปรียบเทียบ

ตัวกรองเหล่านี้จะใช้ตามลำดับที่ระบุในไฟล์นี้ ตัวกรอง action="drop" จะทำให้ยกเว้นการจับคู่ URL ตัวกรอง action="pass" จะทำให้รวมการจับคู่ URL ซึ่งเป็นทางลัดให้กับตัวกรองอื่นๆ ในภายหลังที่อาจจับคู่ด้วยเช่นกัน หากไม่มีตัวกรองที่จะจับคู่กับ URL เลย จะมีการรวม URL ไว้ คุณสามารถสร้างกฎที่ค่อนข้างซับซ้อนได้

ค่าการกระทำเริ่มต้นคือ "drop" ประเภทเริ่มต้นคือ "wildcard"

คุณสามารถแก้ไขหรือลบรายการเหล่านี้ได้ตามความเหมาะสมสำหรับเว็บไซต์ของคุณ อย่างไรก็ตาม รายการตัวอย่างในส่วนนี้จะไม่มีการวางแผนไว้ และอาจมีประโยชน์กับคุณ ไม่เหมือนกับตัวอย่างข้างต้น
********************************************************* -->

<!-- Exclude URLs that end with a '~' (IE: emacs backup files) -->
<filter action="drop" type="wildcard" pattern="*~" />

<!-- Exclude URLs within UNIX-style hidden files or directories -->
<filter action="drop" type="regexp" pattern="/\.[^/]*" />

    คุณสามารถใช้การกรองเพื่อยกเว้น URL ที่ต้องการจาก Sitemap ที่สร้างขึ้น คุณอาจต้องทำเช่นนี้เพื่อสร้างรายการที่สะอาด เพื่อลดรายการที่ซ้ำซ้อน หรือเพื่อป้องกัน URL จากการจัดทำดัชนี โปรดทราบว่าหากคุณใช้ไฟล์ robots.txt เพื่อป้องกัน URL จากการจัดทำดัชนี แม้ว่าจะมีการรวม URL ไว้ใน Sitemap ของคุณ Google จะไม่ค้นหาหรือจัดทำดัชนีให้

    คุณสามารถใช้วิธีการกรองใดๆ หรือทั้งหมด คุณสามารถลบรายการที่คุณไม่ต้องการ และสามารถสร้างรายการเพิ่มเติมได้ตามต้องการ ด้านล่างนี้เป็นตัวอย่างการใช้งาน 

    <filter action="drop" type="wildcard" pattern="*.jpg" />

    ตัวกรองจะยกเว้น URL ที่ลงท้ายด้วย .jpg คุณอาจต้องรวมตัวกรองที่คล้ายกันหากรูปภาพทั้งหมดในเว็บไซต์ของคุณมีการฝังอยู่ภายในหน้าเว็บ HTML และไม่สามารถเข้าถึงได้จาก URL เพียงอย่างเดียว 

    <filter action="pass" type="wildcard" pattern="*.htm*" />
<filter action="drop" type="wildcard" pattern="*" />
    ตัวกรองนี้จะรวมไฟล์ .htm* ทั้งหมดแต่ไม่รวมสิ่งอื่นๆ

การอ้างอิงไวยากรณ์ไฟล์กำหนดค่า

คำอธิบายทั้งหมดของไวยากรณ์ไฟล์กำหนดค่าจะมีอยู่ที่ด้านล่างนี้ แต่ละแท็กจะเริ่มต้นด้วยโค้ดตัวอย่างแล้วตามด้วยคำอธิบายของแอตทริบิวต์

site
แท็กที่จำเป็นในการเริ่มต้นไฟล์กำหนดค่าแต่ละไฟล์ 

<sitebase_url="http://www.example.co.th/"
store_into="/var/www/html/sitemap.xml.gz"
verbose="1"
supress_search_engine_notify="1"
default_encoding="UTF-8">

base_url
 จำเป็น เส้นทาง HTTP ของข้อมูลพื้นฐานในเว็บไซต์ของคุณ - เฉพาะ URL ที่ขึ้นต้นด้วยข้อมูลพื้นฐานเท่านั้นที่จะสามารถรวมอยู่ใน Sitemap 
store_into
 จำเป็น เส้นทางเว็บเซิร์ฟเวอร์ไปยังไฟล์เอาต์พุตที่ต้องการ สคริปต์จะสร้างไฟล์นี้ - ไม่จำเป็นต้องสร้างไฟล์ก่อนเรียกใช้สคริปต์ 
verbose
 ตัวเลือก ป้อนตัวเลข 0-3 โดยที่ตัวเลขมากจะมีข้อมูลการดีบักมากขึ้นด้วย 
suppress_search_engine_notify
 ตัวเลือก ปิดใช้งานการแจ้งเตือนเครื่องมือค้นหาโดยป้อน "1" สำหรับจุดประสงค์ในการทดสอบ 
default_encoding
 ตัวเลือก ระบุการเข้ารหัสอักขระที่จะใช้กับเส้นทางของระบบไฟล์และ URL

url
แท็กตัวเลือกที่คุณสามารถใช้เพื่อแสดง URL แต่ละรายการในเว็บไซต์ของคุณ 

<url href="http://www.example.co.th/stats?q=age" 
lastmod="2004-11-14T01:00:00-07:00" 
changefreq="yearly" 
priority="0.3"
/> 

href
 จำเป็น เส้นทาง HTTP ของข้อมูลพื้นฐานในเว็บไซต์ของคุณ - เฉพาะ URL ที่ขึ้นต้นด้วยข้อมูลพื้นฐานเท่านั้นที่จะสามารถรวมอยู่ใน Sitemap 
lastmod
 ตัวเลือก เวลาที่มีการแก้ไข URL ล่าสุดในรูปแบบ วันที่และเวลาแบบ W3C (ปปปป-ดด-วว เวลา hh:mm:ss+00:00) คุณสามารถละส่วนที่เป็นเวลาได้ ตัวอย่าง:
  "2005-02-21T18:00:15+00:00"
  "2005-02-21"		 
changefreq
 ตัวเลือก ความถี่ในการเปลี่ยนแปลง URL ซึ่งอาจพิจารณาว่าเป็นข้อเสนอแนะและไม่ใช่คำสั่ง ค่าจะต้องเป็นอย่างใดอย่างหนึ่งต่อไปนี้ "ตลอดเวลา" "ทุกชั่วโมง" "ทุกวัน" "ทุกสัปดาห์" "ทุกเดือน" "ทุกปี" หรือ "ไม่เคย" 
priority
 ตัวเลือก ลำดับความสำคัญของหน้าเว็บนี้เมื่อเปรียบเทียบกับหน้าเว็บอื่นๆ ในเว็บไซต์เดียวกัน ค่าจะเป็นตัวเลขระหว่าง 0.0 ถึง 1.0 โดยที่ 0.0 หมายถึงลำดับความสำคัญน้อยที่สุด และ 1.0 หมายถึงลำดับความสำคัญมากที่สุด ลำดับความสำคัญอาจมีผลกับลำดับการเลือก URLของเครื่องมือค้นหาในการสำรวจเว็บไซต์ของคุณ เนื่องจากลำดับความสำคัญมีความเกี่ยวเนื่องกัน และจะใช้เพื่อเลือกระหว่าง URL ต่างๆ ภายในเว็บไซต์ของคุณเอง ลำดับความสำคัญของหน้าเว็บจะไม่เปรียบเทียบกับลำดับความสำคัญของหน้าในเว็บไซต์อื่นๆ

urllist
แท็กตัวเลือกที่คุณสามารถใช้เพื่อชี้ไปยังไฟล์ข้อความที่มีรายการ URL ในเว็บไซต์ของคุณ 

<urllist path="/var/www/html/urllist.txt" encoding="UTF-8" />

path
 จำเป็น เส้นทางและชื่อไฟล์ของไฟล์ .txt คุณสามารถระบุเป็นเส้นทางแบบสัมพัทธ์หรือเส้นทางแบบสมบูรณ์ 
encoding
 ตัวเลือก การเข้ารหัสของไฟล์หากไม่ใช่แบบ UTF-8

ไฟล์ urllist.txt เป็นไฟล์ข้อความแบบง่ายที่มีรายการ URL ที่จะแมป คุณยังสามารถรวมแอตทริบิวต์ตัวเลือกสำหรับ URL แต่ละรายการ โดยต้องป้อนแอตทริบิวต์ในบรรทัดเดียวกันกับ URL และคั่นด้วยช่องว่างเดี่ยว ตัวอย่างเช่น: 

http://www.example.co.th/abc/something
http://www.example.co.th/abc/xyy.pdf lastmod=2001-12-31T14:05:06+00:00
http://www.example.co.th/abc/def?x=12&y=23 changefreq=weekly priority=0.3

priority
 ตัวเลือก ลำดับความสำคัญของหน้าเว็บนี้เมื่อเปรียบเทียบกับหน้าเว็บอื่นๆ ในเว็บไซต์เดียวกัน ค่าจะเป็นตัวเลขระหว่าง 0.0 ถึง 1.0 โดยที่ 0.0 หมายถึงลำดับความสำคัญน้อยที่สุด และ 1.0 หมายถึงลำดับความสำคัญมากที่สุด ลำดับความสำคัญอาจมีผลกับลำดับการเลือก URL ของเครื่องมือค้นหาในการสำรวจเว็บไซต์ของคุณ เนื่องจากลำดับความสำคัญมีความเกี่ยวเนื่องกัน และจะใช้เพื่อเลือกระหว่าง URL ต่างๆ ภายในเว็บไซต์ของคุณเอง ลำดับความสำคัญของหน้าเว็บจะไม่เปรียบเทียบกับลำดับความสำคัญของหน้าในเว็บไซต์อื่นๆ

accesslog
แท็กตัวเลือกที่คุณสามารถใช้เพื่อระบุเส้นทางและชื่อไฟล์ของ IIS และไฟล์บันทึกการเข้าถึงในรูปแบบ Apache เพื่อให้เครื่องมือสร้าง Sitemap สามารถเลือก URL ได้อัตโนมัติ 

<accesslog path="/etc/httpd/logs/access-0.log" encoding="UTF-8"/>
path
 จำเป็น ระบุเส้นทางไปยังไฟล์ 
encoding
 ตัวเลือก ระบุการเข้ารหัสของไฟล์หากไม่ใช่แบบ UTF-8

sitemap
แท็กตัวเลือกที่คุณสามารถใช้เพื่อระบุเส้นทางและชื่อไฟล์ของ Sitemap ที่มีอยู่ ซึ่งคุณสร้างขึ้นด้วยเครื่องมือสร้าง Sitemap เครื่องมือสร้าง Sitemap จะสร้าง Sitemap รายการเดียวที่รวม URL ที่มีใน Sitemap แต่ละรายการ 

<sitemap path="/var/www/docroot/subpath/sitemap.xml" />
path
 จำเป็น ระบุเส้นทางไปยังไฟล์ Sitemap

filter
แท็กตัวเลือกที่คุณสามารถใช้เพื่อสร้างกฎเพื่อรวมหรือยกเว้นไฟล์ที่ต้องการ ตัวกรองจะทำตามลำดับที่ปรากฏในไฟล์ config.xml อย่างไรก็ตาม รายการกรองที่ผสมกัน (url, urllist, directory หรือ accesslog) จะไม่มีผลกระทบเพิ่มเติม URL ทุกๆ รายการที่เครื่องมือสร้าง Sitemap เพิ่มลงใน Sitemap จะได้รับการเปรียบเทียบกับตัวกรองทุกตัวก่อน หากไม่มีตัวกรองใดตรงกับ URL ค่าเริ่มต้นคือ รวม URL ไว้ใน Sitemap

<filter action="drop" type="wildcard" pattern="*/internal/*" />
action
 ตัวเลือก

การกระทำที่ตัวกรองควรดำเนินการ ค่าที่ถูกต้องได้แก่:
  • drop - จะยกเว้น URL ที่ตรงกัน นี่คือการกระทำเริ่มต้น ดังนั้นหากไม่มีการระบุการกระทำเครื่องมือสร้างจะใช้ "drop"
  • pass - จะรวม URL ที่ตรงกัน

 
type
 ตัวเลือก

ประเภทของการกรอง ค่าที่ถูกต้องได้แก่:
  • wildcard - สัญลักษณ์แทนเส้นทางมาตรฐาน (? และ *) จะใช้เพื่อเปรียบเทียบกับ URL นี่คือประเภทเริ่มต้น ดังนั้นหากไม่มีการระบุประเภทเครื่องมือสร้างจะใช้ "wildcard"
  • regexp - การแสดงผลปกติที่จะใช้
 
pattern
 จำเป็น ระบุรูปแบบที่จะจับคู่

การเข้ารหัส

ไฟล์ที่ระบุโดยไฟล์กำหนดค่าของคุณ ไม่ว่าจะเป็นรายการ URL หรือไฟล์บันทึกของเว็บเซิร์ฟเวอร์ก็สามารถใช้การเข้ารหัสอื่นที่ไมใช่แบบ UTF-8 เริ่มต้น คุณสามารถระบุการเข้ารหัสทางเลือกใน config.xml เพื่อให้มีผลกับวิธีการอ่านไฟล์ของเครื่องมือสร้าง Sitemap ต่อไปนี้เป็นการเข้ารหัสทั่วไป:

  • encoding="utf-8" เป็นค่าเริ่มต้น
  • encoding="ascii" คือชุดย่อยของ UTF-8 ดังนั้น คุณจึงไม่ต้องระบุ
  • encoding="iso-8859-1" เป็นแบบทั่วไปสำหรับภาษายุโรปตะวันตก
2b. ข้อมูลเพิ่มเติมสำหรับการสร้างไฟล์กำหนดค่าสำหรับมือถือ

คุณสร้างไฟล์กำหนดค่าสำหรับ Mobile Sitemap ด้วยวิธีเดียวกับ Sitemap ที่ไม่ใช่สำหรับมือถือ อย่างไรก็ตาม คุณต้องสร้างไฟล์กำหนดค่าแยกต่างหากสำหรับภาษามาร์กอัปแต่ละภาษา และเรียกใช้เครื่องมือสร้าง Sitemap พร้อมกับไฟล์กำหนดค่าแต่ละไฟล์แยกกัน เพื่อให้คุณสามารถสร้าง Sitemap ต่างหากสำหรับแต่ละไฟล์

ไฟล์กำหนดค่าแต่ละไฟล์ต้อง:

  • ระบุชื่อไฟล์ที่แตกต่างกันสำหรับ ค่า store_into
  • ใช้ ตัวกรอง เพื่อระบุ URL ที่จะยกเว้นและรวมสำหรับภาษามาร์กอัป โปรดทราบว่า Sitemap แต่ละรายการควรมี URL สำหรับภาษามาร์กอัปเพียงภาษาเดียวเท่านั้น ซึ่งหมายความว่า URL เดียวกันสามารถรวมอยู่ใน Sitemap ต่างๆ ได้ หาก URL เหล่านั้นแสดงภาษามาร์กอัปหลายภาษา

ตัวอย่างของการกรอง

ต่อไปนี้เป็นตัวอย่างวิธีการที่คุณสามารถใช้ตัวกรองส่วนขยายเพื่อสร้าง Mobile Sitemap สำหรับภาษามาร์กอัปต่างๆ คุณควรใช้การกรองเฉพาะอย่างตามประเภทของภาษามาร์กอัปที่ใช้ในเว็บไซต์ของคุณ และวิธีที่คุณระบุแต่ละประเภท หากคุณใช้งานรายละเอียดของเว็บไซต์แตกต่างกัน (ตัวอย่างเช่น คุณอาจจัดระเบียบ URL ด้วยภาษามาร์กอัปต่างๆ ในโฟลเดอร์ต่างหาก) คุณควรกรองตามลักษณะการใช้งานเว็บไซต์ของคุณ โปรดทราบว่าตัวกรองจะใช้ตามลำดับที่คุณแสดงไว้ในไฟล์กำหนดค่า ดังนั้น ตัวกรองแรกที่คุณควรแสดงคือ การกระทำ "pass" ซึ่งระบุ URL ที่คุณต้องการรวมไว้ใน Sitemap

หากต้องการสร้าง Sitemap สำหรับเนื้อหา WML (WAP 1.2):

<filter action="pass" type="wildcard" pattern="*.wml" />
<filter action="drop" type="wildcard" pattern="*.*" />

หากต้องการสร้าง Sitemap สำหรับเนื้อหา XHTML mobile profile (WAP 2.0):

<filter action="pass" type="wildcard" pattern="*.xhtml" />
<filter action="drop" type="wildcard" pattern="*.*" />
3. การอัปโหลดไฟล์ไปยังเว็บเซิร์ฟเวอร์

คุณควรอัปโหลดไฟล์ต่อไปนี้ไปยังเว็บเซิร์ฟเวอร์ของคุณในตำแหน่งที่คุณสามารถเข้าถึงจากบรรทัดคำสั่ง:

  • config.xml —นี่คือไฟล์กำหนดค่าที่คุณเพิ่งสร้างโดยใช้ example_config.xml
  • sitemap_gen.py —นี่คือสคริปต์ Python ที่สร้าง Sitemap ของคุณ
  • urllist.txt —ไฟล์นี้เป็นตัวเลือก ซึ่งคุณต้องรวมไว้เฉพาะในกรณีที่คุณใช้ไฟล์ข้อความในการสร้าง Sitemap

วิธีที่คุณใช้ในการอัปโหลดไฟล์เหล่านี้จะขึ้นอยู่กับระบบของคุณ วิธีทั่วไป ได้แก่ FTP และ SCP สำหรับข้อมูลเพิ่มเติม โปรดติดต่อเว็บโฮสต์ของคุณ

4. การเรียกใช้สคริปต์ของเครื่องมือสร้าง Sitemap (sitemap_gen.py)

ในการเรียกใช้เครื่องมือสร้าง Sitemap คุณจะต้องเชื่อมต่อกับเว็บเซิร์ฟเวอร์ของคุณ วิธีที่คุณใช้ในการเชื่อมต่อจะขึ้นอยู่กับระบบของคุณ ตัวอย่างเช่น โดยทั่วไป คุณสามารถเข้าถึงเซิร์ฟเวอร์ที่ใช้ UNIX ได้โดยใช้ SSH สำหรับข้อมูลเพิ่มเติมเกี่ยวกับการเชื่อมต่อไปยังเว็บเซิร์ฟเวอร์ของคุณ และการเรียกใช้สคริปต์ ให้ปรึกษากับเว็บโฮสต์ของคุณ

หลังจากที่คุณคัดลอกไฟล์ไปยังเว็บเซิร์ฟเวอร์ของคุณแล้ว คุณจะต้องเรียกใช้สคริปต์ของเครื่องมือสร้าง Sitemap เชื่อมต่อกับเว็บเซิร์ฟเวอร์ของคุณและเรียกใช้คำสั่งต่อไปนี้ (แทนที่ <path/config.xml> ด้วยเส้นทางและชื่อไฟล์ของไฟล์กำหนดค่า หากคุณอัปโหลดไฟล์นี้ไปยังตำแหน่งเดียวกันกับสคริปต์ Python คุณสามารถยกเว้นเส้นทางได้):

python sitemap_gen.py --config=<path/config.xml>

5. การส่ง Sitemap ของคุณไปยัง Google

เครื่องมือสร้าง Sitemap จะสร้างไฟล์ sitemap.xml.gz ในตำแหน่งที่คุณระบุในไฟล์กำหนดค่า เมื่อมีการสร้างไฟล์นี้เรียบร้อยแล้ว ตรวจสอบให้แน่ใจว่าสามารถเข้าถึงไฟล์ผ่านเว็บเบราว์เซอร์ได้ จากนั้น เพิ่ม ลงในบัญชี Google Sitemaps ของคุณ ซึ่งจะทำให้ Google สามารถแจ้งสถานะและข้อมูลทางสถิติที่เป็นประโยชน์แก่คุณ หาก Google รายงานปัญหาที่มีใน Sitemap ของคุณ คุณก็สามารถแก้ไขปัญหาและส่ง Sitemap ใหม่ได้ คุณต้องเพิ่ม Sitemap ด้วยตนเองเพียงครั้งเดียวเท่านั้น หลังจากนั้น คุณสามารถใช้ คำขอ HTTP เพื่อแจ้งให้ Google ทราบเกี่ยวกับการเปลี่ยนแปลงใน Sitemap ของคุณ (แม้ว่าคุณสามารถส่ง Sitemap ผ่านบัญชี Google Webmaster Tools ก็ได้)

6. การตั้งค่าสคริปต์ที่เกิดขึ้นประจำ

เราขอแนะนำให้ตั้งค่าเครื่องมือสร้าง Sitemap ให้ทำงานบ่อยเท่ากับความถี่ในการเปลี่ยนแปลงเนื้อหาของคุณ จนถึงบ่อยที่สุดคือชั่วโมงละหนึ่งครั้ง

เว็บมาสเตอร์ที่ใช้เว็บเซิร์ฟเวอร์ UNIX อาจพิจารณาการตั้งค่านี้เป็น งาน cron

เว็บมาสเตอร์ที่ใช้แพลตฟอร์มอื่นๆ ควรติดต่อกับผู้ดูแลระบบของแพลตฟอร์มนั้นๆ เพื่อขอความช่วยเหลือในการกำหนดค่าสคริปต์ที่เกิดขึ้นประจำ นอกจากนี้ คุณอาจได้รับประโยชน์จากคำแนะนำเชิงลึกใน Google Sitemaps Group ที่ Google Groups

คุณสามารถใช้ คำขอ HTTP เพื่อแจ้งให้ Google ทราบเกี่ยวกับการเปลี่ยนแปลงใน Sitemap ของคุณ อย่างไรก็ตาม โปรดตรวจสอบให้แน่ใจว่าคุณเข้าสู่ระบบ Google Webmaster Tools ด้วยบัญชีผู้ใช้ Google เพื่อ เพิ่ม Sitemap ของคุณ ด้วยตนเองในบัญชี Google Webmaster Tools ของคุณ

การแก้ไขปัญหา

ในส่วนนี้เราจะกล่าวถึงคำถามทั่วไปบางคำถามหรือปัญหาบางอย่างที่พบในขณะที่ใช้เครื่องมือสร้าง Sitemap และสิ่งที่คุณสามารถทำได้หากคุณพบปัญหาเหล่านี้

การเข้าถึงเว็บ

ปัญหา: เครื่องมือสร้าง Sitemap แจ้งข้อผิดพลาดและคำเตือนต่อไปนี้:

 
	
	[ERROR] When attempting to access your generated Sitemap at the following URL:
	http://www.example.co.th/sitemap.xml.gz
	we failed to read it.  Please verify the store_into path you specified in
	your configuration file is web-accessible.  Consult the FAQ for more
	information.
	[WARNING] Proceeding to notify with an unverifiable URL.

เกิดอะไรขึ้น เครื่องมือสร้าง Sitemap สร้างไฟล์ที่คุณระบุ จากนั้น พยายามเรียกไฟล์โดยใช้ HTTP แบบเดียวกับที่เครื่องมือค้นหาใช้ แต่ล้มเหลว ซึ่งขั้นตอนก็ดำเนินต่อไปและมีการแจ้งให้เครื่องมือค้นหาทราบ จากนั้นจึงแสดงข้อผิดพลาดและคำเตือนที่แจ้งให้คุณทราบว่า เครื่องมือค้นหาอาจไม่สามารถอ่าน Sitemap ของคุณได้

เครื่องมือสร้าง Sitemap จะสร้างไฟล์ Sitemap ของคุณในเส้นทางที่ระบุในแอตทริบิวต์ store_into ของไฟล์ config.xml จากนั้นเครื่องมือสร้าง Sitemap จะสร้าง URL ให้กับไฟล์นั้นโดยใช้แอตทริบิวต์ base_url และรายงาน URL ไปยังเครื่องมือค้นหา ตัวอย่างเช่น หากคุณกำหนดค่าเป็น:

 
	<site base_url="http://www.example.co.th/"
	      store_into="/var/www/html/sitemap.xml.gz">

เครื่องมือสร้าง Sitemap จะแจ้งให้เครื่องมือค้นหาทำการค้นหา Sitemap ของคุณที่:

	http://www.example.co.th/sitemap.xml.gz

ไฟล์จะต้องสามารถเข้าถึงผ่าน URL นี้ หากไม่สามารถเรียกไฟล์โดยใช้ URL นี้ได้ เครื่องมือค้นหาจะไม่มีทางค้นพบ Sitemap ของคุณ

คุณสามารถทำอะไรได้บ้าง คุณต้องยืนยันว่า config.xml ของคุณระบุเส้นทาง base_url และ store_into ที่ถูกต้องสำหรับ Sitemap ของคุณ นอกจากนี้ คุณต้องยืนยันว่าเว็บเบราว์เซอร์สามารถเรียกไฟล์ Sitemap จาก base_url หากคุณพบข้อผิดพลาดในเส้นทางหรือ URL คุณจะต้องเรียกใช้เครื่องมือสร้าง Sitemap ใหม่หรือเพียงย้ายไฟล์ Sitemap ไปยังตำแหน่งที่ถูกต้อง และแจ้งให้เครื่องมือค้นหาทราบด้วยตนเองผ่านเว็บไซต์ของเครื่องมือค้นหา

การสนับสนุนสำหรับ XML

ปัญหา: เครื่องมือสร้าง Sitemap แจ้งข้อผิดพลาดต่อไปนี้:

 
	
	Some installs of Python 2.2 do not include complete support for XML. 
	Please try upgrading your version of Python and re-running the script.

เกิดอะไรขึ้น เมื่อข้อความระบุว่า บางแพลตฟอร์มมี Python (ภาษาที่ใช้เขียนสคริปต์ของเครื่องมือสร้าง Sitemap) รุ่นที่ไม่มีไลบรารีการสนับสนุนที่จำเป็นสำหรับการประมวลผลไฟล์ XML สคริปต์นี้ต้องมีการสนับสนุน XML อย่างเต็มที่ในการเรียกใช้งาน

คุณสามารถทำอะไรได้บ้าง ลองอัปเกรด Python ที่คุณติดตั้งให้เป็นรุ่นใหม่ คุณอาจจะต้องติดต่อผู้ดูแลระบบของคุณให้ดำเนินการนี้

โปรดทราบว่านี่จะมีผลกับเครื่องมือสร้าง Sitemap เท่านั้น ไม่ใช่ Sitemap ทั้งหมด หากคุณมีวิธีหรือเครื่องมืออื่นในการสร้าง Sitemap คุณสามารถใช้งานและส่ง Sitemap ของคุณไปยังเครื่องมือค้นหา

ไฟล์เพิ่มเติมทั้งหมดใน .gz หรือ .zip คืออะไร

เมื่อคุณแยกสคริปต์ sitemap_gen.py คุณอาจจะเห็นไฟล์เพิ่มเติมมากกว่าที่เรากล่าวถึงข้างต้น ไฟล์ทั้งหมดควรมีลักษณะดังนี้:

 
	
	AUTHORS
	ChangeLog
	COPYING
	example_config.xml
	example_urllist.txt
	PKG-INFO
	README
	setup.py
	sitemap_gen.py
	test_sitemap_gen.py

ไฟล์เพิ่มเติมเหล่านั้นน่าจะเป็นข้อมูลเกี่ยวกับแพคเกจและข้อกำหนดในการอนุญาต ขอแนะนำให้คุณอ่านไฟล์เหล่านี้ด้วย

ข้อยกเว้นอย่างหนึ่งคือ test_sitemap_gen.py ซึ่งเป็นสคริปต์สำหรับการทดลองที่นักพัฒนารายอื่นๆ อาจพบว่าเป็นประโยชน์ หากต้องการช่วยเหลือโครงการโอเพนซอร์สนี้ หากคุณตั้งใจใช้เครื่องมือสร้าง Sitemap โดยไม่แก้ไขรหัสต้นฉบับเลยเช่นเดียวกับผู้ใช้ส่วนใหญ่ คุณก็ไม่จำเป็นต้องใช้สคริปต์ทดลองนี้

หากคุณสนใจให้ความช่วยเหลือโครงการนี้ โปรดไปที่ http://sourceforge.net/projects/goog-sitemapgen โปรแกรมอื่นที่สนับสนุนโพรโทคอล Sitemap

อักขระที่ไม่ใช่แบบ ASCII

ปัญหา: ชื่อโดเมนของเว็บไซต์หรือ URL ของคุณมีอักขระที่ไม่ใช่แบบ ASCII

โดยทั่วไป URL ที่มีอักขระที่ไม่ใช่แบบ ASCII ควรได้รับ การเข้ารหัสแบบ UTF-8 ก่อนที่จะมีการยกเว้นเปอร์เซ็นต์ อย่างไรก็ตาม เว็บเซิร์ฟเวอร์บางแห่งจะทำงานได้อย่างถูกต้องเฉพาะเมื่อ URL ได้รับการเข้ารหัสแบบอื่นๆ ที่ไม่ใช่แบบ UTF-8 เท่านั้น URL ทั้งหมดใน Sitemap ของคุณและ URL ของ Sitemap เองต้องได้รับการเข้ารหัสเพื่อให้เว็บเซิร์ฟเวอร์ที่ URL นั้นวางอยู่สามารถอ่านได้ ภายใน ส่วนคำนิยามของเว็บไซต์ ให้ใช้แอตทริบิวต์ตัวเลือก default_encoding เพื่อระบุการเข้ารหัสที่ใช้โดยเว็บเซิร์ฟเวอร์ของคุณ หากคุณไม่ใช้แท็กนี้และเว็บเซิร์ฟเวอร์ของคุณใช้การเข้ารหัสอื่นๆ ที่ไม่ใช่แบบ UTF-8 เราจะไม่ทราบว่าควรใช้การเข้ารหัสแบบใด

หาก URL ของคุณมีอักขระที่ไม่ใช่แบบ ASCII เราขอแนะนำให้คุณเรียกใช้สคริปต์ของเครื่องมือสร้าง Sitemap โดยใช้ Python 2.3 ขึ้นไป Python รุ่นนี้ได้เพิ่มการสนับสนุนการเข้ารหัสที่ไม่ใช่แบบ ASCII หากชื่อโดเมนของคุณมีอักขระที่ไม่ใช่แบบ ASCII คุณต้องใช้ Python 2.3 ขึ้นไป เนื่องจากการสนับสนุน Internationalizing Domain Names in Applications (IDNA) จะไม่มีอยู่ในรุ่นเก่า หากไม่มีการสนับสนุน IDNA เครื่องมือสร้าง Sitemap จะไม่สามารถเข้ารหัสชื่อโดเมนที่ไม่ใช่แบบ ASCII ได้อย่างถูกต้อง
